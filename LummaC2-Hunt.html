Executive Summary

Primary - Red Canary Red Canary identifies LummaC2, also called Lumma Stealer, as a Windows infostealer sold on underground forums since at least August 2022. In Red Canary's 2025 Threat Detection Report it ranked seventh by prevalence, affecting 3.9 percent of monitored customers and rising from rank 22 in the prior year.

Primary - Red Canary The Red Canary page emphasizes credential and sensitive-data theft from browsers, browser extensions, email applications, two-factor authentication material, and cryptocurrency wallets. Its most durable detection opportunities are suspicious fodhelper.exe execution from script interpreters and mshta.exe launching hidden PowerShell download cradles.

Enrichment Microsoft, FBI/CISA, CIS, MITRE ATT&CK, and later campaign reporting add infrastructure, delivery, persistence, and collection details: fake CAPTCHA / ClickFix paste-run flows, C2 domains, /c2sock HTTP exfiltration, OpenWith.exe UAC bypass, Chrome remote debugging on port 9222 to bypass app-bound encryption, file staging under eight-hex-character directories, and injected collection processes such as dllhost.exe, RegAsm.exe, msbuild.exe, and regsvcs.exe.

Hunt Brief & Attack Chain

LummaC2 operations are best hunted as a chain instead of a static IOC list: user-facing lure, script or LOLBin execution, local privilege or evasion maneuver, credential collection, short-lived staging, and outbound exfiltration.

Stage Primary-source facts Enrichment / adjacent pivots Defender decision
Delivery Red Canary lists malicious ads, fake software downloads, SEO poisoning, malicious CAPTCHA sites, phishing, drive-by downloads, Discord phishing messages, and legitimate GitHub account abuse. Microsoft, CIS, and Bitdefender describe fake CAPTCHA / ClickFix paste-run PowerShell, cracked software, AI/deepfake lures, Steam profile payload hosting, blockchain-hosted payloads, and CastleLoader delivery. Hunt browser-to-script transitions, copied PowerShell cradles, newly downloaded CPL/EXE payloads, and unusual archive extraction followed by script execution.
Execution Red Canary highlights mshta.exe launching PowerShell with bypass, no-profile, hidden-window, and web-download patterns. Observed commands use curl.exe, temp CPL/EXE paths, Start-Process, and staged PowerShell variables. Alert when mshta.exe, browsers, or Explorer spawn hidden PowerShell that contacts the internet or writes into user-writable paths.
Privilege / UAC bypass Red Canary highlights unexpected fodhelper.exe under script interpreters. Microsoft and CIS describe OpenWith.exe plus HKCU\Software\Classes\ms-settings\shell\open\command and an empty DelegateExecute value. Track both UAC bypass families and treat registry changes plus scheduled task creation as high-confidence triage anchors.
Collection Red Canary describes theft of browser credentials, extensions, email data, 2FA data, and cryptocurrency wallets. Microsoft describes System.txt, user_data, System.zip, injected dllhost.exe, and Chrome remote debugging on port 9222. Hunt non-browser process access to browser databases, staging file creation, and chrome.exe --remote-debugging-port=9222.
C2 / exfiltration Red Canary notes a May 2025 Microsoft and law-enforcement takedown and reduced sightings afterward. FBI/CISA and Microsoft list C2 domains, IPs, and HTTP POSTs to /c2sock; Bitdefender reports later resurgence activity through CastleLoader. Block confirmed current infrastructure, hunt aged infrastructure as context, and pivot to rare-domain POSTs from suspicious process trees. 
Attack chain sketch:
User lure -> fake CAPTCHA / malvertising / phishing / cracked software
  -> browser, Explorer, or mshta launches PowerShell / curl / CPL / EXE
  -> fodhelper.exe or OpenWith.exe UAC bypass attempt
  -> injected dllhost.exe / RegAsm.exe / msbuild.exe / regsvcs.exe
  -> browser, wallet, 2FA, email, and local file collection
  -> System.txt / user_data / System.zip staging
  -> HTTP POST to Lumma C2, often observed with /c2sock in enrichment reporting

Affected Surface & Telemetry

Endpoint process telemetry

Required for the highest-confidence hunts: process creation, parent-child ancestry, command line, file hash, script interpreter launch, and network-enriched process events.

File and registry telemetry

Needed for staging path, Chrome debug artifact, UAC bypass registry, scheduled task, and browser credential store access coverage.

DNS, proxy, firewall, and EDR network telemetry

Needed for C2 domains, IPs, rare-domain HTTP POSTs, and /c2sock pivots. Preserve both domain and process context where available.

Identity, SaaS, and cloud telemetry

Needed after suspected theft: new sessions, token refresh, impossible travel, MFA changes, password-manager access, and developer platform activity.

Browser and application policy telemetry

Chrome remote debugging, app-bound encryption bypass attempts, downloaded file provenance, and enterprise browser extension inventory can validate impact.

High-risk users and hosts

Prioritize administrators, developers, finance users, password-manager users, browser-saved credential users, and endpoints with local admin rights.

Native Audit-Log Hunts

Windows process creation

Search Event ID 4688, Sysmon Event ID 1, or EDR process logs for mshta.exe -> powershell.exe, script interpreter -> fodhelper.exe, OpenWith.exe launched by task infrastructure, and chrome.exe --remote-debugging-port=9222.

Registry and scheduled tasks

Search Sysmon 12/13/14, Windows Task Scheduler logs, and endpoint registry telemetry for HKCU\Software\Classes\ms-settings\shell\open\command, empty DelegateExecute, and \Microsoft\Windows\RemoteApp and Desktop Connections Update\.

File artifacts

Search Sysmon 11, EDR file events, and forensic triage for eight-hex-character staging directories under C:\ProgramData and AppData\Local, System.txt, System.zip, user_data, and Chrome debug directories under temp.

DNS and proxy

Search DNS, secure web gateway, firewall, and proxy logs for enrichment domains, enrichment IPs, rare-domain POSTs after script execution, and URIs containing /c2sock.

Identity and SaaS

For every suspected endpoint, review password resets, MFA changes, new device registrations, new OAuth grants, token refreshes, unusual geography, and developer platform access for the impacted user.

Browser and password stores

Audit enterprise browser policy, local password storage, remote debugging settings, extension inventory, and suspicious file access to browser profile databases.

Detection Validation Gates

Gate Required telemetry Positive validation Benign control Promotion rule
Red Canary execution behaviors Process creation with parent, command line, and hash. Atomic or lab execution of mshta -> powershell and script interpreter -> fodhelper. Approved HTA app and endpoint management scripts. Promote IOA-1 and IOA-2 after seven days of zero unmanaged false positives.
OpenWith UAC bypass Registry, process, and scheduled task telemetry. Create test HKCU ms-settings key and task in an isolated lab. Normal Windows OpenWith shell usage and legitimate scheduled tasks. Detect high by default; prevent after helpdesk and admin baseline.
Infrastructure pivots DNS, proxy, firewall, and EDR network process context. Controlled lookup or synthetic event for listed domains and IPs. Known sinkhole, threat-intel scanner, or sandbox traffic. Block current high-confidence domains; hunt-only aged IPs and generic URI pivots.
Browser credential theft File read/open telemetry or EDR sensitive file telemetry. Authorized test process reads browser profile files in a lab. Backup software, browser, password manager, and EDR scanner access. Use host risk scoring until non-browser false positives are understood.
CQL compatibility Falcon LogScale dataset with Falcon event fields. Run each card in a 7-day window and confirm no parser errors. Field-not-found results are acceptable during parser mapping. Save tuned tenant-specific variants with local field names and lookup tables.

Hardening

  • Restrict or monitor mshta.exe, wscript.exe, cscript.exe, regsvr32.exe, rundll32.exe, and script interpreters from launching payloads in user-writable directories.
  • Use WDAC, AppLocker, or equivalent policy to block unsigned executables, CPL files, and script payloads from Downloads, temp, and user profile paths.
  • Enable PowerShell Script Block Logging, Module Logging, and Transcription where privacy and volume allow; enforce Constrained Language Mode for non-admins where compatible.
  • Disable or tightly control Chrome remote debugging on managed endpoints, especially port 9222. Audit developer exceptions separately.
  • Reduce browser-stored credential risk through enterprise password managers, passkeys, phishing-resistant MFA, and policies that limit password saving where feasible.
  • Set UAC to "Always notify" for high-risk users and remove unnecessary local administrator rights.
  • Block known malicious domains/IPs after currentness checks; add DNS filtering for newly registered, suspicious, and uncategorized domains.
  • Harden web download paths with SmartScreen, file reputation, archive scanning, and warning controls for fake CAPTCHA / paste-run lures.
  • Protect developer workstations with stricter secrets handling, browser profile controls, GitHub token rotation procedures, and CLI credential monitoring.
  • Train users that CAPTCHA pages should not ask them to press Windows+R, paste a command, or run PowerShell.

Falcon response actions

Run RTR collection for process list, autoruns, scheduled tasks, suspect files, registry key exports, browser policy state, and DNS cache. Quarantine suspicious staged files only after hash and path capture.

Identity response

For every impacted user, revoke sessions, rotate browser-saved passwords, reset password-manager master sessions where applicable, rotate developer tokens, review OAuth grants, and enforce step-up MFA.

Network response

Block current enrichment C2 domains and IPs after recency validation. Add detections for HTTP POSTs to rare domains from hosts that also show script execution, UAC bypass, or browser file access.

Containment Runbook

  1. Isolate the endpoint in Falcon or the EDR control plane. Preserve volatile process, network, and command-line evidence before cleanup.
  2. Run Q18 to find related hosts, then pivot through Q1-Q17 for process, registry, file, hash, DNS, and network confirmation.
  3. Collect staged artifacts from C:\ProgramData, %LOCALAPPDATA%, temp Chrome debug directories, scheduled task exports, and the ms-settings registry key.
  4. Kill confirmed malicious processes and remove malicious scheduled tasks and registry values after evidence capture.
  5. Rotate credentials for the impacted user: browser-saved passwords, password manager sessions, email, VPN, cloud, SaaS, developer tokens, GitHub tokens, API keys, and cryptocurrency wallet secrets if present.
  6. Revoke active sessions and refresh tokens. Review identity logs for activity after the first suspected theft timestamp.
  7. Block confirmed current C2 domains, IPs, and hashes. Keep aged indicators in hunt-only lists with expiry notes.
  8. Reimage endpoints with confirmed credential theft or unknown persistence. Validate no re-created UAC registry keys, scheduled tasks, remote debugging flags, or staging paths remain.
  9. Run a 14-day watch: new sign-ins, password changes, MFA resets, suspicious OAuth grants, and outbound traffic from previously impacted users and hosts.
