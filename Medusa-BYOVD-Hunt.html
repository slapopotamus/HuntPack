Must-know caveat: The Medusa BYOVD step is the single highest-leverage detection point in the kill chain. If a Medusa-family driver (smuol.sys, mbaer.sys, nitrogenk.sys, etc.) successfully loads, the EDR agent will likely be terminated or callback-blinded within seconds. Kill Chain

  1. Initial Access (T1190): Exploit unpatched public-facing Microsoft Exchange (most common) or use credentials supplied by an Initial Access Broker. Symantec notes ~40% of investigated attacks could not have initial access determined.
  2. Discovery (T1018, T1087): quser, net user, NetScan, PDQ Inventory.
  3. Credential Access (T1003.003): Access to ntds.dit, Mimikatz / Impacket against LSASS. Spearwing creates a VSS shadow copy first (vssadmin create shadow /for=C:) then mounts.
  4. Lateral Movement / Persistence (T1021.001, T1219): RDP, PsExec, plus heavy use of RMM tools dropped into user Documents: SimpleHelp (mx.exe, sn.exe), AnyDesk, Mesh Agent (mesh.exe -fullinstall), RevSocks (svhost.exe).
  5. Mass-Distribution (T1569.002): PDQ Deploy + PDQ Inventory. Files deployed from CSIDL_WINDOWS\AdminArsenal\PDQDeployRunner\Service-1\exec\ — Symantec observed this same path in ~⅔ of Medusa attacks.
  6. THE BYOVD STEP (T1543.003 + T1068 + T1553.002): Kernel driver loaded as Windows service from non-System32 path. ABYSSWORKER signed with stolen Chinese cert; POORTRY attestation-signed; classical BYOVD legitimately signed. Service registers at HKLM\System\CurrentControlSet\Services\<svc>\ImagePath pointing at .sys in \Users\<user>\Documents\ or \Windows\Temp\.
  7. EDR Kill (T1562.001 + T1014): User-mode loader sends DeviceIoControl to the driver. ABYSSWORKER unregisters kernel callbacks via IOCTL 0x222400 (blinding EDR), then terminates EDR processes via IOCTL 0x222144. Userland fallback uses taskkill /F /IM <edr>.exe when BYOVD unavailable.
  8. Exfiltration (T1567): Renamed Rclone (lsp.exe) to mega.nz / gofile.io / anonfiles.com / S3 generics with --config, --no-check-certificate, --transfers; FRP frpc.exe for persistent tunneling.
  9. Inhibit Recovery (T1490): vssadmin delete shadows.
  10. Encryption Impact (T1486): gaze.exe deployed via PDQ Deploy. Skips .dll, .exe, .lnk, .MEDUSA and folders WindowsOld, Perflogs, Msocache, ProgramFiles, ProgramFilesX86, Programdata. AES-256; drops !READ_ME_MEDUSA!!!.txt recursively; self-deletes (-d arg disables).

HUNT-2 services.exe loading driver from non-System32 path (generic BYOVD staging)
MITRE: T1543.003 / T1068 FP Risk: Low Confidence: High Timeframe: 30 days retro; 7-day rolling Requires: Falcon DriverLoad + ProcessRollup2

Hypothesis: Legitimate Windows drivers live in \System32\drivers\. Any services.exe parent loading a .sys from \Users\*\Documents\, \Windows\Temp\, \ProgramData\, or \AppData\ is highly likely a BYOVD staging event. Catches all three Medusa driver families regardless of hash rotation.

FP notes: Anti-cheat (EAC, BattlEye), Razer/Logitech/HP/Dell/Lenovo HW vendor tools install drivers from \ProgramData\<Vendor>\. Vendor-exclusion filter handles these; treat unsigned drivers from these paths as P1 regardless.

// HUNT-2: services.exe loading driver from non-System32 path (generic BYOVD staging)
// MITRE: T1543.003 Service Install + T1068 PrivEsc via driver
// HYPOTHESIS: services.exe parent + .sys image-load from user-writable dir == BYOVD staging
// FP RISK: Low - high-fidelity behavioral pattern
// FP NOTES: Anti-cheat (EAC, BattlEye), some HW vendor tools. Exclusions applied below.
// CONFIDENCE: High
// TIMEFRAME: 30 days retro; 7-day rolling
#event_simpleName = DriverLoad
| ImageFileName = /\\(Users\\[^\\]+\\(Documents|Downloads|AppData)|Windows\\Temp|ProgramData)\\.*\.sys$/i
| !(ImageFileName = /\\ProgramData\\(Microsoft|Intel|NVIDIA|AMD|EasyAntiCheat|BattlEye|Razer|Logitech|HP|Dell|Lenovo)\\/i)
| !(AuthenticodeIssuerName = /Microsoft (Windows|Code Signing PCA|Corporation)/i)

MITRE: T1569.002 / T1543.003 FP Risk: Medium Confidence: High Timeframe: 30 days retro Requires: Falcon ProcessRollup2

Hypothesis: Spearwing uses PDQ Deploy in ~⅔ of Medusa attacks. The chain pdqdeployrunner-1.exe → (gaze.exe | *.sys | known KillAV-named loaders) is a Spearwing fingerprint.

HUNT-4 Userland EDR/AV kill commands (taskkill / net stop / sc stop / wmic / pskill)
MITRE: T1562.001 FP Risk: Medium Confidence: High Timeframe: 7 days rolling Requires: Falcon ProcessRollup2

Hypothesis: Userland fallback EDR-kill — Spearwing's secondary technique when BYOVD fails. A single PID issuing ≥2 EDR-kill commands within 60 seconds is high-fidelity.

FP notes: EDR vendor upgrade scripts may legitimately stop services. Filter on vendor-signed installers via AuthenticodeSubjectName. BYOVD-driven (kernel IOCTL) kills are not directly observable from process telemetry — HUNT-1/2/5/6 are the primary defense for that path.

// MITRE: T1562.001 Impair Defenses: Disable or Modify Tools

Strategic (broader attack-surface reduction — 1 to 3 months)

ST-1 — WDAC Application Control — Enforce Mode

Promotion of DF-1 to enforcement after 14-day audit. Build supplemental policies for legitimate drivers/applications. Requires pilot rings, change-management sign-off, documented rollback. M1038, M1045, CIS Win11 18.9.46.x (Level 2).

ST-2 — WDAC / AppLocker Publisher Allow-List for RMM Tools

Defines explicit publisher allow-list (e.g., ScreenConnect, ConnectWise, Splashtop, Atera) and blocks all others. Pair with the CrowdStrike Custom IOA in HUNT-7 for immediate coverage while WDAC is rolled out. M1038, M1042, CIS Win11 2.3.10.x.

ST-3 — Tier 0/1/2 Admin Separation + Privileged Access Workstations

Defeats Spearwing's full kill chain — most steps require Domain Admin or local admin. Implement Microsoft's Securing Privileged Access (SPA) Enterprise Access Model: Tier 0 admins log on only to dedicated PAWs (enforced via Authentication Silos), never to Tier 1/2 endpoints. M1026 (Privileged Account Management), M1018 (User Account Management).

Why this kills Spearwing: their typical post-exploitation move is to dump Domain Admin credentials from a workstation via Mimikatz/ntds.dit, then use those credentials to PsExec/PDQ-Deploy to every endpoint. With Tier 0 admins logging in only on PAWs, those credentials are never present on a Spearwing-compromised workstation in the first place.

ST-4 — Windows LAPS for Local Admin Password Rotation

Built into Windows 11 22H2+ and Server 2019+; replaces the older LAPS toolkit. Configure HKLM\Software\Microsoft\Policies\LAPS → BackupDirectory=2 (AD), PasswordComplexity=4, PasswordLength=20, PasswordAgeDays=30. M1027, M1026; CIS Win11 5.x.

ST-5 — Network Segmentation: Deny SMB/RDP Between Non-Server Workstations

PDQ Deploy uses SMB (ADMIN$) shares and RPC on port 445. Workstation-to-workstation SMB/RDP traffic is rarely legitimate; deny by default at the Windows Defender Firewall (Domain Profile) or perimeter NGFW. Pair with ST-3. M1037, M1030; CIS Win11 9.2.x.

ST-6 — Egress Filtering on Common Cloud-Storage Exfil Destinations

Block at perimeter: *.mega.nz, *.mega.io, *.mega.co.nz, *.gofile.io, *.anonfiles.com, *.bayfiles.com, *.pixeldrain.com, *.file.io, *.transfer.sh, *.ufile.io, Tor exits, and non-business S3 endpoints. Pair with DLP rule alerting on >1 GB outbound to non-corporate destinations in a 60-minute rolling window. M1037, M1057.

ST-7 — Credential Guard

Registry: HKLM\SYSTEM\CurrentControlSet\Control\LSA → LsaCfgFlags = 1 (UEFI lock) or 2 (no lock, for pilot). Verify via Win32_DeviceGuard.SecurityServicesRunning — must include 1 (Credential Guard) and 2 (HVCI). M1043; CIS Win11 18.9.11.3 (Level 1).

ST-8 — Block PsExec and Living-off-the-Land Lateral-Movement Tools

ASR Rule GUID D1E49AAC-8F56-4280-B9BA-993A6D77406C — Block process creations originating from PSExec and WMI commands. Block mode. Caveat: blocks legitimate PsExec usage; pair with ST-3 so IT admins use SCCM/Intune remoting instead. M1038, M1040.

ST-9 — Disable Service Auto-Load for Drivers Not in DriverStore

Built into HVCI (IH-1) on HVCI-enabled hosts. For non-HVCI environments, enforce via WDAC supplemental policy requiring drivers be sourced from \SystemRoot\System32\DriverStore\. M1054, M1038.

ST-10 — Disable WDigest Credential Caching (Legacy Hygiene)

Registry: HKLM\SYSTEM\CurrentControlSet\Control\SecurityProviders\WDigest → UseLogonCredential = REG_DWORD 0. Lowest-effort hygiene control in this document; should already be set in any modern environment. M1027; CIS Win11 18.9.27.1.1 (Level 1).

The remaining nineteen controls expand the surface area and bring the environment into alignment with Microsoft Security Baselines and CIS Level 1/2 — durable improvements regardless of which RaaS operator we're concerned with next.
