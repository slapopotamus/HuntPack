Threat Writeup Summary

Source: Elastic Security Labs — MIMICRAT: Custom RAT Mimics Multiple C2 Frameworks

URL: https://www.elastic.co/security-labs/mimicrat-custom-rat-mimics-c2-frameworks

Campaign Overview

Elastic Security Labs documented a ClickFix-delivered campaign that ends in a previously-unreported native-C++ RAT they dubbed MIMICRAT. The infection chain abuses the user-paste-from-clipboard trick (Win+R / clipboard PowerShell), patches ETW + AMSI in memory, then loads a statically-linked Lua 5.4.7 interpreter to decrypt and execute a Meterpreter-family shellcode that finally stages the MIMICRAT beacon. MIMICRAT's network profile is deliberately built to look like traffic from other known C2 frameworks (forged User-Agent, mimicked URIs, Google referer) — "mimic" in the name. Targets observed include a US university and Chinese-speaking users; the ClickFix lure is localized into 17 languages and rides on compromised legitimate sites (bincheck.io, investonline.in).

The Attack Chain (Step-by-Step)

  1. ClickFix lure: Victim browses a compromised legit site (e.g., bincheck.io); injected JavaScript from investonline.in/js/jq.php renders a fake Cloudflare "verify you are human" page that instructs the victim to press Win+R and paste a clipboard payload, or paste into PowerShell directly.
  2. Stage 1 — Obfuscated PowerShell one-liner: Launched with -WInDo Min (windowstyle minimised). A garbled variable like $RdLU='aZmEwGEtHPckKyBXPxMRi.neTwOrkicsGf' is sliced/transformed at runtime to reconstruct the C2 domain (xMRi.neTwOrk → 45.13.212.250). This avoids plaintext IOCs in process telemetry.
  3. Stage 2 — ETW + AMSI bypass: Downloaded PowerShell patches System.Diagnostics.Eventing.EventProvider.m_enabled to 0 (kills script-block logging) and flips System.Management.Automation.AmsiUtils.amsiInitFailed to true. Marshal.Copy rewrites the ScanContent method bytes for belt-and-braces AMSI neutralization.
  4. Stage 3 — Lua 5.4.7 loader (zbuild.exe): A native PE staged at %ProgramData%\knz_{random}\zbuild.exe embeds the Lua interpreter and an encrypted Lua script. Decrypts via fxh::utility::lua_script_xor_decrypt, custom-alphabet Base64-decodes, then maps shellcode in-memory — fileless from this point.
  5. Stage 4 — Meterpreter-family shellcode: Standard reverse-HTTPS reflective loader; pulls the final beacon from the post-exploitation C2 tier (www.ndibstersoft.com, 23.227.202.114).
  6. Stage 5 — MIMICRAT beacon: 64-bit native C++ implant. C2 hostname (d15mawx0xveem1.cloudfront.net) is RC4-encrypted in the binary; session crypto is RSA-1024 + AES (hardcoded IV abcdefghijklmnop, key = runtime SHA-256). HTTPS to TCP/443, 10-second callback interval. 22-command dispatch table covering token theft (CMD 31), shellcode injection (CMD 100), SOCKS5 (CMD 101), interactive shell over pipe (CMD 78), and full file/process manipulation.
  7. Network mimicry: HTTP profiles forge Edge/Cortana User-Agents, a Google-search Referer, and Cookie names (AFUAK, BLA, HFK; POST: ARCHUID, BRCHD, ZRCHUSR) designed to look like traffic from common C2 frameworks (Cobalt Strike, Sliver, etc.) — defeating analysts who hunt for "framework X" rather than the actual TTPs.

Why It Matters

  • ClickFix bypasses email gateways entirely. Delivery is via compromised legit websites + clipboard social-engineering. Attachment filtering doesn't apply; the user types the payload themselves.
  • The PowerShell is invisible by default. ETW + AMSI patches land within seconds, before defenders ever see the script content. Detection has to catch execution — the explorer.exe → powershell.exe parentage from the Run dialog, or the Lua loader path — not the script body.
  • Mimicked C2 traffic. Hunting on "Cobalt Strike-style" or "Sliver-style" indicators will misroute investigators. MIMICRAT is identifiable by its combination of forged artifacts (Cortana UA + zh-CN Accept-Language + Google referer + specific URIs).
  • Custom Base64 + Lua = signature evasion. The Lua script is XOR-encrypted with a non-standard Base64 alphabet, so static signature engines miss the staged payload.
  • Token theft built in. CMD 31 duplicates tokens by PID; CMD 12 spawns processes under stolen credentials. Lateral movement is one operator command away.

Hunt Brief

  • Threat: MIMICRAT / ClickFix campaign High Severity
  • Summary: ClickFix-style social engineering tricks users into pasting an obfuscated PowerShell one-liner via Win+R. The script patches ETW + AMSI, drops a Lua 5.4.7 loader (zbuild.exe) under %ProgramData%\knz_*\, decrypts and runs a Meterpreter-family shellcode, and stages the MIMICRAT native-C++ beacon to CloudFront over TCP/443 with HTTP traffic crafted to look like other C2 frameworks.
  • Affected: Windows endpoints (x64). Documented victims include a US university and Chinese-speaking users; the ClickFix lure is localized into 17 languages so geo-targeting should be assumed wide.
  • Actor: Unattributed; capability profile consistent with an experienced operator (custom RAT, multi-stage in-memory chain, C2 mimicry).
  • CISA KEV: No (TTP-based campaign, not a CVE)

Hunt 12 — Token Theft: Process Spawn With Different User Context Than Parent
MITRE: T1134.001 · FP Risk: Med-High · Confidence: Medium
// HUNT: MIMICRAT CMD 31 + CMD 12 duplicate process tokens and spawn under stolen credentials
// HYPOTHESIS: A child process running under a different user/SID than its parent indicates token theft, UNLESS
//             the parent is a known service-context binary that legitimately re-impersonates (svchost, services, etc.)
// FP RISK: Med-High - this pattern is common for legitimate elevation (UAC, RunAs, scheduled tasks). Use as a hunt prompt, not an alert.
// FP NOTES: ParentUserSid / ParentUserName may not be populated in every PR2 record - LogScale repos vary. Baseline
//           your own field availability first. Heavy tuning required - widen the parent exclusion list as you observe
//           legitimate cases in your environment. Focus on rows where parent is a USER-context process spawning a
//           child running as a DIFFERENT interactive user (not SYSTEM/LOCAL SERVICE/NETWORK SERVICE).
// CONFIDENCE: Medium (requires analyst review)
// TIMEFRAME: Last 14 days

#event_simpleName = /ProcessRollup2|SyntheticProcessRollup2/
// Exclude well-known system parents that legitimately spawn under a different SID
| ParentBaseFileName != /^(services|svchost|lsass|wininit|smss|csrss|TrustedInstaller|MsMpEng|MoUsoCoreWorker|RuntimeBroker|sihost|fontdrvhost|spoolsv|conhost|userinit|dwm|taskhostw|ctfmon|searchindexer|consent|UsoClient|backgroundTaskHost|SearchProtocolHost|SearchFilterHost|SgrmBroker)\.exe$/i
// Require BOTH SID and UserName mismatch (defends against the field being partially populated)
| UserSid != ParentUserSid
| UserName != ParentUserName
// Require both sides populated and neither side a service account (these are noisy and legitimate)
| UserName != "" AND ParentUserName != ""
| !in(field=UserName, values=["SYSTEM", "LOCAL SERVICE", "NETWORK SERVICE"], ignoreCase=true)
| !in(field=ParentUserName, values=["SYSTEM", "LOCAL SERVICE", "NETWORK SERVICE"], ignoreCase=true)
// Aggregate so analysts see frequency, not raw rows
| groupBy([aid, ComputerName, ParentBaseFileName, ImageFileName, ParentUserName, UserName], function=collect([@timestamp, CommandLine], limit=20))
| sort(_count, order=desc)

Hardening Recommendations

Immediate — Target MIMICRAT / ClickFix TTPs

1. Kill ClickFix at the Win+R Run-Dialog

  • GPO: User Configuration → Administrative Templates → Start Menu and Taskbar → "Remove Run menu from Start Menu" for non-admin users.
  • AppLocker / WDAC rule: deny powershell.exe and cmd.exe when parented by explorer.exe AND command line contains http / iwr / IEX.
  • User-awareness module focused on "paste this into Win+R to verify you're human" prompts. This is the single biggest cultural fix.

2. Constrain PowerShell + Force Visibility

  • Deploy Constrained Language Mode via WDAC for non-admin users — the ETW/AMSI bypass relies on .NET reflection that CLM blocks.
  • Enable Script Block Logging (4104), Module Logging (4103), Transcript Logging: HKLM\SOFTWARE\Policies\Microsoft\Windows\PowerShell\ScriptBlockLogging\EnableScriptBlockLogging = 1.
  • Forward 4104 to SIEM and alert on script content matching the AMSI/ETW bypass strings (Hunt 5).
  • Disable PowerShell v2: Disable-WindowsOptionalFeature -Online -FeatureName MicrosoftWindowsPowerShellV2*.
  • Block powershell.exe -ExecutionPolicy Bypass/Unrestricted via WDAC custom rule where feasible.

3. Application Control on %ProgramData% and %AppData%

  • WDAC / AppLocker: deny unsigned PE execution under %ProgramData%\, %AppData%\, %LocalAppData%\Temp\, %Public%\. This breaks the Lua loader stage directly.
  • Where full deny is too aggressive, enforce a publisher allow-list (Microsoft + your sanctioned vendors).

4. Browser / Web Defense

  • Deploy a content-security web proxy (Zscaler, Netskope, Cloudflare Gateway, etc.) and block clipboard-injection script categories.
  • Block known ClickFix landing patterns at proxy level (regex on URL path: /js/jq.php, cloudflare.*verify, verify[-_]captcha).
  • Web filtering: increase scrutiny for newly-observed domains in .cc, .network, .top, .icu, .xyz when accessed from corporate endpoints.

5. Token-Theft Mitigation

  • Enable LSA Protection (RunAsPPL=1) and Credential Guard; both blunt CMD 31 (token duplication) by limiting the token surface accessible to user-mode processes.
  • For tier-0 admins, enforce Protected Users group membership.

6. Block the C2 Infrastructure

  • Firewall / DNS block xmri.network, wexmri.cc, ndibstersoft.com (and www.), d15mawx0xveem1.cloudfront.net, 45.13.212.250-251, 23.227.202.114.
  • URL block: /js/jq.php path on investonline.in at proxy only; do not block the parent domain (legitimate compromised site).
  • Do NOT block *.cloudfront.net wholesale — massive shared infrastructure.

Strategic — Reduce Attack Surface Generally

  • Egress filtering: deny direct internet egress from endpoints on TCP/80 and TCP/443 to non-corporate destinations; force through proxy with TLS inspection where legally feasible.
  • WDAC enforcement: roll WDAC in audit→enforce mode for all knowledge-worker endpoints. Block unsigned .NET assemblies in user-writable paths.
  • HTTP telemetry into LogScale: if you don't currently forward proxy / NDR HTTP logs into LogScale, Hunt 8 (MIMICRAT URI / Cookie / UA profile) is the highest-fidelity detection you're missing.
  • Clipboard hardening: some environments can deploy clipboard-monitoring agents that flag command-like content copied from non-trusted origins. Combined with Win+R restrictions, this neuters ClickFix.
  • User training cadence: ClickFix is a 2024-2026 staple. Update phishing simulations to include browser-based "paste this to continue" scenarios; HR + finance + dev + research are the highest-yield audiences.
