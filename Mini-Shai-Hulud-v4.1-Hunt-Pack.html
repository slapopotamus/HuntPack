1. Executive Summary

Mini Shai-Hulud is an autonomous, self-replicating npm/PyPI worm operated by TeamPCP — the same actor behind the March 2026 Aqua Trivy and April 2026 Bitwarden CLI compromises. The May 11, 2026 wave (tracked as CVE-2026-45321, CVSS 9.6) used a chained pull_request_target "Pwn Request" → GitHub Actions cache poisoning → in-memory OIDC token theft sequence to publish 84 malicious versions across 42 @tanstack/* packages in six minutes, then propagated via stolen credentials to UiPath, OpenSearch, Mistral AI, Guardrails AI, Squawk, TallyUI, AntV, and SAP CAP namespaces.

Critically, this is the first npm supply-chain attack carrying valid SLSA Build Level 3 provenance attestations — provenance alone is not a sufficient trust signal. The defender priority is preventing stolen credentials from being used to access repositories, publish more poisoned packages, modify CI workflows, steal source code, or pivot into cloud environments.

7. CrowdStrike Custom IOA Recommendations

Deployment guidance: Do not create one giant IOA. Use a small set of high-confidence Custom IOAs for campaign-specific behaviors, keep broad package-manager behavior in Detect until baselined, and use Custom IOCs for exact hashes/domains/IPs. Suggested path: Detect → tune → Prevent, except for exact campaign artifacts and workflow names, which are safe to move to Prevent faster.

Tier 1 — Prevent candidates

  • Confirmed payload artifacts and service names
  • Known C2/payload infrastructure
  • Named malicious GitHub Actions workflow files
  • Exact campaign dead-drop strings

Tier 2 — Detect first

  • Package manager install lifecycle spawning shell/downloader/interpreter
  • Package-manager process reading high-value developer secrets
  • CI runner dependency install followed by GitHub/npm/C2 behavior
  • IMDS access from package-manager process tree

Tier 3 — Hunt / scheduled search

  • Compromised namespace installs without payload artifacts
  • Any workflow YAML write by developer tools
  • GitHub API repo/workflow manipulation from scripted context
  • Session messenger traffic where business use is unknown
