1. Executive Summary

MuddyWater is a persistent Iranian state-sponsored APT assessed with high confidence to be a subordinate element of Iran's Ministry of Intelligence and Security (MOIS). Active since at least 2017, the group targets government, telecom, defense, oil & gas, and financial organizations across the Middle East, Asia, Europe, and North America. USCYBERCOM publicly attributed the group in 2022; CISA Advisory AA22-055A remains active.

The group's operational signature is PowerShell at every stage — from initial execution through C2 to exfil — combined with systematic abuse of legitimate Remote Monitoring and Management (RMM) tools (Atera, SimpleHelp, ScreenConnect, Syncro, NetBird, ConnectWise) as C2 infrastructure, deliberately chosen to blend into normal IT traffic. Delivery is consistently spearphishing using PDF/HTA attachments or links to trojanized RMM installers. Recent evolution (2024–2026) added BugSleep, the Phoenix v4 backdoor, and the Rust-based RustyWater RAT — signals the group is investing in custom tooling to reduce reliance on detectable RMM software. DLL sideloading via a renamed GoogleUpdate.exe alongside a malicious goopdate86.dll is a confirmed persistence mechanism.

2025–2026 evolution. In October 2025 Group-IB documented a Phoenix v4 campaign run from compromised mailboxes (accessed over NordVPN) against 100+ MENA government entities, with VBA-macro Word lures. In December 2025 ESET disclosed a new toolset — the MuddyViper backdoor delivered by the Fooder reflective loader (which masquerades as the classic Snake game and uses a Snake-logic delay routine plus the modern CNG crypto API to evade sandboxes), the VAX One backdoor (impersonates Veeam/AnyDesk/Xerox/OneDrive-updater), and a stack of credential stealers (CE Notes, LP Notes, Blub). Separately, the UDPGangster backdoor (Dec 2025, targeting Turkey/Israel/Azerbaijan) breaks from HTTP C2 entirely — it beacons over raw UDP/1269 to 157.20.182.75, persists by copying itself to %AppData%\RoamingLow\SystemProc.exe with an Explorer\User Shell Run value, and carries heavy anti-VM/anti-debug checks. RMM delivery now spans Atera, SimpleHelp, ScreenConnect, N-able, Action1, PDQ and Level, staged from OneHub, Egnyte, Mega, Sync, TeraBox, filetransfer.io and even Zendesk/Zopim chat uploads. ESET also assesses MuddyWater may act as an initial-access broker for other Iran-aligned groups (overlap with Lyceum/OilRig).

The highest-value defensive angles are: catching Office apps or browsers spawning mshta.exe / scripting engines (earliest delivery signal), detecting browser-parented RMM installer launches (spearphishing link execution), auditing all unauthorized RMM agents fleet-wide (ongoing access), and catching obfuscated PowerShell download cradles (POWERSTATS/BugSleep loader). The certutil -urlcache and bitsadmin /transfer patterns are low-noise high-confidence IOCs.

Defender priority: Run Q6 (RMM inventory) and Q7 (browser-parented RMM install) immediately — any hit on Q7 is an isolate-and-IR trigger. Run Q13/Q14 (DNS/IP IOC sweeps, now including UDPGangster 157.20.182.75) and Q18 (anomalous UDP C2) as a retroactive 90-day sweep. Add Q16 (Run-key persistence) and Q17 (execution from RoamingLow/Public staging) to catch UDPGangster/Fooder, and Q19 (scripting-engine reach-out to file-sharing services) to catch the current RMM delivery vector. Any host touching the known C2 domains/IPs should be treated as actively compromised.

8. CrowdStrike LogScale CQL Hunt Queries

Validate #event_simpleName, ParentBaseFileName, ImageFileName, CommandLine, and field names against your Falcon tenant schema before promoting detections. All process queries use /ProcessRollup2|SyntheticProcessRollup2/ to avoid missing long-running processes.

Q1. Mshta Executing Remote HTA Payload — T1218.005

CONF: high | FP: low | COST: low | Requires: ProcessRollup2 + CommandLine

Looks for: mshta.exe with an HTTP/S URL in CommandLine — MuddyWater's HTA payload fetch. Accomplishes: Catches delivery at the earliest executable moment before C2 check-in.

FP: Some enterprise apps use mshta.exe for local HTA UI — these never contain remote http/https URLs. Very low FP expected.
// HUNT: MuddyWater - T1218.005 Mshta Remote HTA Execution
// HYPOTHESIS: MuddyWater delivers HTA payloads via mshta.exe fetching from remote URLs.
// FP RISK: Low — mshta.exe has no legitimate need to fetch from internet URIs in managed envs
// FP NOTES: Rare enterprise HTA apps use local/intranet UNC paths, never http://
// CONFIDENCE: High
// TIMEFRAME: 30 days recommended; extend to 90 for retroactive sweep

#event_simpleName=/ProcessRollup2|SyntheticProcessRollup2/
| ImageFileName=/\\mshta\.exe$/i
| CommandLine=/https?:\/\//i
| groupBy(
    [ComputerName, UserName, aid],
    function=[count(as=HitCount), max(@timestamp, as=LastSeen), collect([CommandLine, ParentBaseFileName])]
  )
| sort(HitCount, order=desc)

Q2. Regsvr32 Scriptlet (Squiblydoo) — T1218.010

CONF: high | FP: low | COST: low | Requires: ProcessRollup2 + CommandLine

Looks for: regsvr32.exe loading remote .sct scriptlets or scrobj.dll — the Squiblydoo technique used by MuddyWater for COM-based payload delivery. Accomplishes: Catches scriptlet-based execution that bypasses AppLocker script rules.

Q3. Obfuscated PowerShell C2 / Loader — T1059.001 + T1027

12. Hardening — Tiered & Deployable

Immediate (MuddyWater-specific)

  • Block mshta.exe outbound network access. Prevent mshta.exe from establishing internet connections via Windows Firewall outbound rule. AppLocker: deny mshta.exe execution for user contexts if HTA apps are not required. This single control stops HTA payload delivery cold.
  • Audit all RMM agent installations fleet-wide. Run Q6 and Q7 immediately. Any RMM agent not in your authorized inventory = investigate and remove. Block unauthorized MSI package installation via Software Restriction Policy or WDAC.
  • Enable PowerShell Script Block Logging + Transcription. GPO Path: Computer Config → Admin Templates → Windows Components → Windows PowerShell. Enable "Turn on Script Block Logging" + "Turn on PowerShell Transcription." Output to a monitored UNC share. This forces in-memory PowerShell to disk — catches POWERSTATS/BugSleep loader even after obfuscation.
  • Block macro execution from internet-sourced Office documents. GPO: User Config → Admin Templates → Microsoft Office → Security Settings → Block macros from running in Office files from the Internet = Enabled. This blocks the primary MuddyWater VBA macro delivery chain.
  • Restrict certutil and bitsadmin outbound access. Windows Firewall outbound rules blocking certutil.exe and bitsadmin.exe from reaching internet IPs. In managed environments neither tool has a legitimate need to reach internet URIs.

Near Term

  • Restrict regsvr32 remote scriptlet execution. Windows Firewall outbound: block regsvr32.exe from internet access. AppLocker: deny cmstp.exe for non-admin users.
  • Deploy Credential Guard. Prevents LSASS memory dumping by Mimikatz. GPO: Computer Config → Admin Templates → System → Device Guard → Turn On Virtualization Based Security. Requires modern hardware + Secure Boot.
  • Enable LSA Protection (RunAsPPL). Registry: HKLM\SYSTEM\CurrentControlSet\Control\Lsa → RunAsPPL = DWORD 1. Blocks procdump64-based LSASS access.
  • Disable WPAD + LLMNR + NBT-NS. Prevents credential theft via poisoning used in MuddyWater lateral movement. Disable LLMNR via GPO: Computer Config → Admin Templates → Network → DNS Client → Turn Off Multicast Name Resolution = Enabled.
  • Enforce MFA + conditional access on all remote access and email — especially OWA/M365. MuddyWater registers Atera free-trial agents and sends lures from compromised mailboxes (often via leaked/reused OWA credentials). MFA + impossible-travel/legacy-auth blocking directly counters this. Priority: OWA/M365, VPN, RMM console access.
  • Application-control RMM tools (allowlist). Permit only your sanctioned RMM agent (by publisher/path/hash) via WDAC or AppLocker; block all others (Atera, SimpleHelp, ScreenConnect, Syncro, NetBird, Level, PDQ, Action1, N-able, AnyDesk) from executing. This neutralizes the group's primary C2 delivery vector regardless of lure.
  • Block execution from user-writable staging dirs. WDAC/AppLocker path-deny rules for %AppData%\RoamingLow\, C:\Users\Public\, and %Temp% stop UDPGangster (SystemProc.exe) and Fooder loaders from running.

Strategic

  • Implement WDAC (Windows Defender Application Control) in audit mode progressing to enforcement to restrict unsigned script/DLL execution. Directly counters DLL sideloading.
  • Deploy email sandboxing / attachment detonation at the gateway (Proofpoint, Defender for Office 365 Safe Attachments). MuddyWater's primary delivery vector is email — detonating attachments before delivery catches the delivery stage with no endpoint risk.
  • Continuously monitor for new RMM tool installations via automated alerting on Q6 results delta — alert on any new RMM agent process seen that wasn't in the prior 90-day baseline.
  • Web-proxy category control for unsanctioned file-sharing. Block or step-up-authenticate access to free file-sharing services (Egnyte, OneHub, Mega, Sync, TeraBox, filetransfer.io, freeupload.store) for endpoints that have no business need — this breaks the current RMM-installer delivery chain (Q19).
  • Egress-filter non-standard outbound UDP. Default-deny outbound UDP except to approved ports/destinations at the perimeter firewall; this blocks UDPGangster's raw UDP/1269 C2 (Q18) and similar non-HTTP channels.
