Threat Writeup Summary

Source: Sophos X-Ops — QEMU abused to evade detection and enable ransomware delivery

URL: https://www.sophos.com/en-us/blog/qemu-abused-to-evade-detection-and-enable-ransomware-delivery

Campaign Overview

Sophos X-Ops has tracked two related intrusion sets — STAC4713 (Payouts King ransomware, attributed to GOLD ENCOUNTER, active since Nov 2025) and STAC3725 (CitrixBleed2-led intrusions, active since Feb 2026) — that both abuse the open-source QEMU hardware emulator to launch hidden Linux virtual machines on compromised Windows hosts. The VM acts as a covert toolchain platform: AdaptixC2, Havoc, Impacket, BloodHound, Kerbrute, NetExec, Chisel, Rclone, and other offensive tooling all execute inside Alpine Linux guests — outside the visibility of any host-based EDR agent. The host sees only one signed-binary process: qemu-system-x86_64.exe.

The Attack Chain (Step-by-Step)

  1. Initial Access: Either (a) exposed SonicWall SSL-VPN without MFA, (b) CVE-2025-26399 SolarWinds Web Help Desk RCE, or (c) CVE-2025-5777 (CitrixBleed2) NetScaler memory disclosure leaking session tokens.
  2. Foothold: Stage a ZIP (an.zip) containing a dropper (an.exe) plus a ScreenConnect MSI installer. an.exe creates the service AppMgmt and a rogue local admin (CtxAppVCOMService).
  3. RMM Persistence: Install ConnectWise ScreenConnect via MSI — relay points to attacker-controlled vtps.us. ScreenConnect = signed, well-known RMM — blends with legitimate IT traffic.
  4. QEMU Stage: Download qemu_custom.zip containing qemu-system-x86_64.exe and a pre-built Alpine Linux disk image. STAC4713 disguises the disk image as vault.db or bisrv.dll to defeat extension-based scanning; STAC3725 uses custom.qcow2.
  5. Stealth Boot: A scheduled task named "TPMProfiler" launches QEMU under SYSTEM with -display none / -nographic — no GUI, no console, no taskbar entry. QEMU forwards a host port (32567 or 22022) to guest port 22.
  6. Reverse SSH Tunnel: Inside the VM, BusyBox SSH (or Chisel) initiates an outbound reverse tunnel to attacker SSH listeners at 74.242.216.76, 194.110.172.152, 98.81.138.214, or 158.158.0.165. WireGuard with wg-obfuscator wraps the traffic to defeat protocol fingerprinting.
  7. Post-Exploitation (inside VM, invisible to EDR): AdaptixC2 (STAC4713) or Havoc (STAC3725 — sideloaded via ADNotificationManager.exe + vcruntime140_1.dll). Operators run Impacket, NetExec, BloodHound, Kerbrute, Coercer, KrbRelayx against the internal network.
  8. Credential Theft: WDigest registry tweak to force plaintext storage. Volume Shadow Copy abuse to dump NTDS.dit, SAM, SYSTEM hives. Browser cookie/credential extraction via cookie_exporter.exe.
  9. BYOVD & AV Evasion: Vulnerable K7 Security kernel driver K7RKScan_1516.sys loaded for AV/EDR tampering. FTK Imager and Defender exclusion edits prep the host for ransomware staging.
  10. Ransomware Deployment (STAC4713): Payouts King ransomware pushed laterally over SMB; exfiltration via Rclone over the WireGuard tunnel before encryption.

Why It Matters

  • The VM is an EDR blind spot. Falcon (and every other host-based agent) cannot inspect processes, network sockets, or file I/O inside a QEMU guest. The whole toolkit is invisible.
  • One signed binary hides the entire campaign. Defenders see qemu-system-x86_64.exe — an open-source, legitimately signed emulator — running as SYSTEM. Block-listing QEMU is the only real control.
  • Disk images masquerade as ordinary files. A 2 GB vault.db or bisrv.dll on disk doesn’t flag content scanners; only file-magic / size heuristics catch it.
  • Reverse SSH + WireGuard + obfuscator defeats DPI. Egress controls that allow outbound TCP/443 or TCP/22 to arbitrary destinations will pass this traffic.
  • The TTP is generalizing. Beyond Payouts King, the same pattern has been reused with Havoc C2 (STAC3725) and earlier Kaspersky-documented data-exfiltration cases — expect more actors to copy it.

Hunt Brief

  • Threat: QEMU Hidden-VM Abuse (STAC4713 / Payouts King & STAC3725) Critical Severity
  • Summary: Adversaries deploy qemu-system-x86_64.exe as a SYSTEM-level scheduled task to boot a hidden Alpine Linux VM containing the full offensive toolkit. The VM establishes a reverse-SSH/WireGuard tunnel and runs C2 agents (AdaptixC2, Havoc) entirely outside EDR visibility. Used as a delivery platform for Payouts King ransomware and broad credential-theft operations.
  • Affected: Internet-facing Windows servers running SonicWall SSL-VPN (no MFA), SolarWinds Web Help Desk (CVE-2025-26399), Citrix NetScaler (CVE-2025-5777). Lateral movement targets any Windows endpoint reachable over SMB.
  • Actor(s): GOLD ENCOUNTER (Payouts King — STAC4713); attribution pending for STAC3725.
  • CISA KEV: Yes — CVE-2025-5777 (CitrixBleed2) and CVE-2025-26399 are both listed.

Hunt 5 — QEMU Headless / Hidden VM Command-Line Flags
MITRE: T1564.006, T1027 · FP Risk: Low · Confidence: High · Timeframe: 90d
// HUNT: QEMU launched with display-hiding and port-forward flags (covert VM signature)
// HYPOTHESIS: STAC4713/3725 launch QEMU with -display none / -nographic / -daemonize and hostfwd port redirection to forward to guest SSH (port 22)

Hunt 8 — Rogue Service "AppMgmt" / "CtxAppVCOMService"
MITRE: T1543.003, T1136.001 · FP Risk: Low · Confidence: High · Timeframe: 90d
// HUNT: STAC3725 creates the service "AppMgmt" and the local admin "CtxAppVCOMService"
// HYPOTHESIS: an.exe dropper installs persistence via sc.exe / net user; service binary path is non-canonical
// FP RISK: Low - Microsoft's real service is "Application Management" with internal name "AppMgmt" BUT its ImagePath is svchost.exe -k netsvcs
// FP NOTES: Filter out the genuine svchost-hosted AppMgmt service; alert when the binary path is NOT svchost
// CONFIDENCE: High
// TIMEFRAME: Last 90 days

(
  (#event_simpleName = /ServiceStarted|WindowsServiceInstalled/ AND (
    (ServiceDisplayName = /AppMgmt/i AND ImagePath != /\\svchost\.exe/i)
    or ServiceDisplayName = /CtxAppVCOMService/i
  ))
  or
  (#event_simpleName = /ProcessRollup2|SyntheticProcessRollup2/ AND (
    CommandLine = /sc(\.exe)?\s+(create|config).*(AppMgmt|CtxAppVCOMService)/i
    or CommandLine = /net(\.exe)?\s+user\s+CtxAppVCOMService.*\/add/i
  ))
)
| timestamp := @timestamp
| table([timestamp, aid, ComputerName, UserName, ImageFileName, CommandLine, ServiceDisplayName, ImagePath])
| sort(timestamp, order=desc)

Hunt 11 — WDigest UseLogonCredential Set to 1
MITRE: T1003.001, T1112 · FP Risk: Low · Confidence: High · Timeframe: 90d
// HUNT: WDigest plaintext password storage re-enabled
// HYPOTHESIS: Operators flip HKLM\System\CurrentControlSet\Control\SecurityProviders\WDigest\UseLogonCredential = 1 to harvest cleartext creds on next logon
// FP RISK: Low - this key has no legitimate reason to be set to 1 on modern Windows
// FP NOTES: Some legacy compatibility scripts may set it; require justification
// CONFIDENCE: High
// TIMEFRAME: Last 90 days

#event_simpleName = /AsepValueUpdate|RegGenericValueUpdate|RegistryOperationDetectInfo/
| RegObjectName = /\\Control\\SecurityProviders\\WDigest$/i
| RegValueName = /UseLogonCredential/i
// UseLogonCredential is a DWORD - the live value lands in RegNumericValue; some shells write it as a string, so check both
| (RegNumericValue != 0 or RegStringValue = /^[^0]/)
| timestamp := @timestamp
| table([timestamp, aid, ComputerName, UserName, RegObjectName, RegValueName, RegNumericValue, RegStringValue])
| sort(timestamp, order=desc)
Hunt 12 — Volume Shadow Copy Abuse for NTDS / SAM Dumping
MITRE: T1003.003, T1003.002, T1006 · FP Risk: Medium · Confidence: Med-High · Timeframe: 60d
// HUNT: vssadmin / wmic shadowcopy create followed by NTDS.dit / SAM / SYSTEM access
// HYPOTHESIS: Operators dump DC credential stores via shadow-copy to bypass file locks
// FP RISK: Medium - backup software (Veeam, Acronis, Windows Server Backup) uses VSS routinely
// FP NOTES: Tune to exclude known backup service accounts; alert when paired with NTDS/SAM path access
// CONFIDENCE: Medium-High
// TIMEFRAME: Last 60 days

#event_simpleName = /ProcessRollup2|SyntheticProcessRollup2/
| (
    (CommandLine = /(vssadmin|wmic)/i AND CommandLine = /shadowcopy.*create/i)
    or CommandLine = /(ntdsutil|esentutl).*ntds/i
    or CommandLine = /copy.*\\\\.+\\HarddiskVolumeShadowCopy.*\\(NTDS\\NTDS\.dit|System32\\config\\(SAM|SYSTEM|SECURITY))/i
    or CommandLine = /reg(\.exe)?\s+save\s+HKLM\\(SAM|SYSTEM|SECURITY)/i
  )
| timestamp := @timestamp
| table([timestamp, aid, ComputerName, UserName, ParentBaseFileName, ImageFileName, CommandLine])
| sort(timestamp, order=desc)

Hardening Recommendations

Immediate — Target QEMU Hidden-VM TTPs

1. Block unsanctioned hypervisors via WDAC / AppLocker

  • WDAC publisher rule: deny any unsigned binary named qemu-system-*.exe, qemu-img.exe, qemu-nbd.exe, qemu-ga.exe from running outside an explicit VirtLab OU.
  • AppLocker fallback (XML): add a Publisher condition deny for "QEMU Project" signed binaries on knowledge-worker endpoints; allow only via group exemption.
  • Block via path: deny execution from C:\ProgramData\, C:\Users\*\AppData\, and C:\Windows\Temp\ for any binary whose internal product name contains "QEMU".
  • If KVM/Hyper-V/VMware Workstation are also unauthorized, broaden the rule to a virtualization-tool category block.

2. Block .qcow2 / large opaque blob writes in user-writable paths

  • File Server Resource Manager (FSRM) on Windows file servers: file-screen rule blocking *.qcow2, *.vhdx, *.vmdk, *.img writes outside designated virtualization shares.
  • EDR custom rule: alert on any single-file write > 500 MB into C:\ProgramData, C:\Windows\Temp, or %APPDATA% whose extension is .db or .dll (the masquerade pattern).
  • Periodically scan endpoints for files matching qcow2 magic bytes (51 46 49 FB) regardless of file extension.

3. Lock down outbound SSH from servers

  • Windows Defender Firewall (GPO): Computer Configuration → Windows Settings → Security Settings → Windows Defender Firewall with Advanced Security → Outbound Rules → deny TCP/22 to any non-RFC1918 destination from server endpoints.
  • Block the specific reverse-tunnel ports observed in the campaign (TCP/32567, TCP/22022) at the perimeter.
  • Where outbound SSH is required for legitimate admin, force it through a jump-host whose IP is the only allowed destination.

4. Tighten scheduled-task creation visibility

  • GPO: enable Audit Policy → Object Access → Audit Other Object Access Events (success+failure) for scheduled-task changes — generates Event ID 4698/4702.
  • Forward 4698 to your SIEM and alert on any task created under SYSTEM context that runs an executable located outside %SystemRoot% or a signed-vendor install path.
  • Set a baseline of expected SYSTEM scheduled tasks per host; alert on net-new entries.

5. Lock the WDigest registry path

  • Enforce HKLM\System\CurrentControlSet\Control\SecurityProviders\WDigest\UseLogonCredential = 0 via GPO (re-applied each refresh).
  • Apply a DACL deny ACE on the WDigest subkey for non-SYSTEM principals using Group Policy Preferences → Windows Settings → Registry.
  • Pair with LSA Protection (RunAsPPL=1) and Credential Guard.

6. Block the C2 / relay infrastructure

  • Perimeter firewall / DNS sinkhole: block vtps.us, 144.208.127.190, 74.242.216.76, 194.110.172.152, 98.81.138.214, 158.158.0.165.
  • Submit hashes to Falcon IOC Management as Prevent (see CSV above).

7. Patch the initial-access CVEs

  • CVE-2025-5777 (CitrixBleed2): upgrade NetScaler ADC / Gateway to the build listed in the Citrix advisory and rotate all session tokens / cookies issued from vulnerable builds.
  • CVE-2025-26399 (SolarWinds Web Help Desk): apply the SolarWinds hotfix; pull the Help Desk web tier behind an IP allow-list during the patch window.
  • SonicWall SSL-VPN: enforce MFA on every SSL-VPN account; rotate any creds whose accounts pre-date the MFA enforcement date.

Strategic — Reduce Attack Surface Generally

  • Application allow-listing maturity: WDAC in enforce mode for all servers. The QEMU TTP fails completely if unsigned/non-vendor binaries cannot execute.
  • RMM governance: inventory every authorized ScreenConnect / AnyDesk / TeamViewer / Atera install. Network-block all RMM relay traffic except sanctioned relays. Require certificate pinning to your tenant.
  • Service-creation alerting: Windows Event 7045 forwarded to SIEM; alert on services with ImagePath under ProgramData\, AppData\, or Temp\, or whose display name shadows a Microsoft service but whose binary is not svchost.exe.
  • Driver allow-listing: enable Microsoft Vulnerable Driver Blocklist (Memory Integrity / Core Isolation) on every endpoint — blocks K7RKScan and the broader BYOVD class.
  • Egress filtering: deny direct internet egress from servers to arbitrary IPs/ports. Force everything through a logged, identity-aware proxy. The reverse-SSH tunnel breaks if outbound 22 / arbitrary high-ports are denied.
  • Asset inventory of disk images: baseline what .qcow2 / .vhdx / .img / .vmdk files should exist on each host; alert on any unbaselined image.
  • LSA Protection + Credential Guard across the fleet — blunts WDigest, LSASS-dumping, and DPAPI attacks.
  • Domain controller hardening: protect NTDS.dit with VSS-aware monitoring; alert on vssadmin/wmic shadowcopy create followed by file reads from HarddiskVolumeShadowCopy*.
