Threat Writeup Summary

Source: Cyfirma Research — Operation SilentCanvas: JPEG-Based Multistage PowerShell Intrusion

URL: https://www.cyfirma.com/research/operation-silentcanvas-jpeg-based-multistage-powershell-intrusion/

Campaign Overview

Cyfirma researchers documented a multi-stage intrusion campaign that weaponizes a fake .jpeg file to deliver PowerShell, achieve fileless privilege escalation, and establish persistent command-and-control through a trojanized ConnectWise ScreenConnect client. The attack is assessed as the work of a financially motivated actor or Initial Access Broker demonstrating professional engineering and operational maturity.

The Attack Chain (Step-by-Step)

  1. Delivery: Victim receives sysupdate.jpeg via phishing, fake cloud-sharing link, or spoofed software update lure. The file lacks valid JPEG magic bytes (FF D8 FF) — it is PowerShell with a misleading extension.
  2. Execution: Double-click launches PowerShell with -ExecutionPolicy unrestricted; a staging directory C:\Systems is created.
  3. Stage 2 download: Script fetches access.jpeg and runs it via Invoke-Expression (in-memory, fileless). AMSI is bypassed through triple-layer string obfuscation (quote-splitting, format strings, runtime reconstruction).
  4. On-host compilation: Microsoft's csc.exe compiler is invoked to build a custom launcher (uds.exe) from temporary C# source (runner.cs). Each compile produces a unique hash, defeating signature-based detection.
  5. Fileless UAC bypass: Attacker writes to HKCU:\Software\Classes\ms-settings\Shell\Open\Command with an empty DelegateExecute value, pointing to ../../Systems/uds.exe. Launching ComputerDefaults.exe (auto-elevated, no UAC prompt) triggers the payload at elevated privilege. Registry keys are deleted ~2 seconds later to destroy forensic evidence.
  6. Persistence: sc.exe creates a Windows service named OneDriveServers (masquerading as legitimate Microsoft OneDrive) that runs ScreenConnect.ClientService.exe from C:\ProgramData\OneDriveServer\. Hidden local administrator accounts are created with showInLogon=false.
  7. Environment prep: Hidden .cmd scripts in C:\Windows\SystemTemp\ScreenConnect\ use DISM.exe to silently enable .NET Framework 3.5; WMI queries against root\SecurityCenter2 enumerate installed antivirus.
  8. C2: Trojanized ScreenConnect establishes encrypted channels to legitserver.theworkpc.com (45.138.16.64) over TCP/5443 (download) and TCP/8041 (C2). Uses custom PBKDF2/HMAC-SHA256 session key derivation and an internal DNS cache to evade network defenses.
  9. Post-exploitation: Full remote-access kit — screen capture/recording, microphone & speaker audio capture, clipboard interception, keylogging via SetWindowsHookEx, credential harvesting via Credential Provider hooking and named-pipe IPC, file transfer via encrypted virtual streams, token impersonation, and Safe Mode reboot to disable security tools.

Why It Matters

  • Living-off-the-land throughout. Every binary executed (powershell.exe, csc.exe, cvtres.exe, ComputerDefaults.exe, DISM.exe, sc.exe) is signed by Microsoft. No malicious binary needs to land on disk in a way AV can fingerprint — the launcher is compiled fresh per host.
  • Signed C2. ScreenConnect is a legitimate, code-signed RMM product. Network defenders see "ConnectWise" traffic and may pass it without scrutiny.
  • The UAC bypass is fileless. Registry-only, self-cleaning, no scheduled task or autorun to find with traditional IR triage.
  • The fake JPEG bypasses extension-only email filtering. Many gateways trust .jpeg; this file is PowerShell that happens to end in .jpeg.
  • Credential Provider hooking captures credentials at the Windows logon prompt — before LSASS, before any password manager.

Hunt Brief

  • Threat: Operation SilentCanvas High Severity
  • Summary: Multi-stage PowerShell intrusion using a .jpeg-extension loader (no JPEG magic bytes) that performs AMSI bypass, on-host compiles a launcher via csc.exe, executes a fileless UAC bypass via the ms-settings COM-handler hijack, then deploys a trojanized ConnectWise ScreenConnect as a service named OneDriveServers for persistent C2.
  • Affected: Windows endpoints with PowerShell + .NET; especially organizations running RMM tooling (ScreenConnect / AnyDesk / TeamViewer).
  • Actor: Unattributed — assessed as financially motivated / Initial Access Broker.
  • CISA KEV: No (TTP-based campaign, not a CVE)

  • TCP ports: 5443 (payload), 8041 (C2)

Hardening Recommendations

Immediate — Target SilentCanvas TTPs

1. Disable double-click execution of script content masquerading as images

  • Group Policy: User Configuration → Administrative Templates → Windows Components → Attachment Manager → "Do not preserve zone information" → Disabled.
  • Enforce HKCU\Software\Microsoft\Windows\CurrentVersion\Explorer\FileExts\.jpeg and .jpg → OpenWithList to image viewers only. Remove any references to txtfile or Microsoft.PowerShellScript.1.
  • Block PowerShell from being a registered handler for non-script extensions.

2. Constrain PowerShell

  • Deploy Constrained Language Mode via WDAC/AppLocker for non-admin users.
  • Enable Script Block Logging (Event 4104), Module Logging (4103), and Transcript Logging: HKLM\SOFTWARE\Policies\Microsoft\Windows\PowerShell\ScriptBlockLogging\EnableScriptBlockLogging = 1.
  • Block powershell.exe -ExecutionPolicy Bypass/Unrestricted via WDAC custom rule where feasible.
  • Disable PowerShell v2 engine: Disable-WindowsOptionalFeature -Online -FeatureName MicrosoftWindowsPowerShellV2*

3. Block / monitor csc.exe and cvtres.exe

  • AppLocker / WDAC: deny csc.exe, cvtres.exe, vbc.exe, jsc.exe execution outside developer OUs.
  • At minimum, alert on any execution under a non-Administrator account.

4. Close the ms-settings UAC bypass

  • Set HKLM\SOFTWARE\Microsoft\Windows\CurrentVersion\Policies\System\ConsentPromptBehaviorAdmin = 2 (Always Prompt on the secure desktop).
  • Set EnableLUA = 1 and PromptOnSecureDesktop = 1.
  • Deploy UAC level "Always Notify" via GPO for privileged users.
  • Deny HKCU\Software\Classes\ms-settings writes via WDAC/MDAC ruleset.

5. RMM Governance
