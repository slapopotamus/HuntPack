Executive Summary

"The Gentlemen" is a professionalized double-extortion ransomware group active in 2026. They focus on mass-exploitation of perimeter network appliances (FortiOS, FortiProxy, SonicWall SSL VPN, Cisco ASA) as initial access, then pivot to internal AD reconnaissance, deploy commodity Remote Monitoring & Management (RMM) tools for persistence (AnyDesk, ScreenConnect, Atera, Splashtop), tamper with endpoint defenses, delete shadow copies, and finally encrypt with double-extortion data theft.

Exploit-to-Encrypt Chain (typical)

  1. External: Appliance exploited from internet. Operator harvests creds/sessions from the appliance.
  2. T+0–4h: Authenticates to internal network via the VPN as a legitimate-looking user from a VPN-pool IP.
  3. T+1–24h: Runs AD reconnaissance (net.exe, nltest, dsquery, AdFind, SharpHound).
  4. T+24–48h: Deploys at least one RMM tool (AnyDesk preferred) for persistence that survives VPN session revocation.
  5. T+48h+: Lateral movement (RDP, SMB, WMI), credential theft, defense tamper, then encryption with mass shadow-copy deletion.

Endpoint-Observable Indicators (the hunt focus)

  • Interactive logons (LogonType 10) originating from VPN concentrator-assigned IPs to non-jump-host endpoints
  • net.exe / net1.exe enumerating "Domain Admins", "Enterprise Admins"
  • nltest /dclist:, /domain_trusts, /domain
  • SharpHound.exe / Invoke-BloodHound / collection-method strings; *_computers.json, *_groups.json file writes
  • AnyDesk.exe / ScreenConnect.ClientService.exe / AteraAgent.exe written to C:\ProgramData\ or %TEMP%
  • vssadmin delete shadows, wmic shadowcopy delete, bcdedit /set recoveryenabled no
  • Set-MpPreference -DisableRealtimeMonitoring $true, sc stop WinDefend, taskkill /IM MsMpEng.exe

Q1 AD Discovery — Domain Admin / DC / Trust enumeration
FP RISK: MEDIUM CONF: MEDIUM
MITRE: T1087.002, T1069.002, T1482, T1018
Timeframe: Last 7 days
Requires: Falcon Pro+

Hypothesis: Post-VPN-compromise, operators enumerate privileged groups, domain controllers, and trusts using built-in tools before moving laterally.

FP Notes: Sysadmin scripts, AD health checks, JIT/PAW workstations. Allow-list known admin AIDs after triage.

↗ Open in Falcon 
// HUNT: The Gentlemen - T1087.002 / T1069.002 / T1482 / T1018 - AD Discovery
// HYPOTHESIS: Post-VPN-compromise operators run built-in AD enumeration tooling
//             before they move laterally and dump credentials.
// FP RISK: Medium - sysadmin and tier-1 helpdesk legitimately run these
// FP NOTES:
//   - net.exe by domain admins from PAW/JIT workstations
//   - nltest.exe inside AD health-check scripts on member servers
//   - dsquery from documented PowerShell modules
// CONFIDENCE: Medium - behavioral pattern; tune by allow-listing admin AIDs
// TIMEFRAME: Last 7 days

#event_simpleName = /ProcessRollup2|SyntheticProcessRollup2/
| in(field=FileName, values=["net.exe","net1.exe","nltest.exe","dsquery.exe","whoami.exe","adfind.exe"], ignoreCase=true)
| CommandLine = /(group\s+"?Domain Admins"?|group\s+"?Enterprise Admins"?|\/domain\b|\/dclist|\/domain_trusts|\/groups|\/all|enterprise admins|domain admins)/i
| groupBy([aid, ComputerName, UserName],
    function=[count(), collect([ImageFileName, CommandLine], limit=20)])
| sort(_count, order=desc)

Q5 ⭐ VPN-pool logon → AD recon within 60 min (correlated)
FP RISK: MEDIUM CONF: HIGH
MITRE: T1078 + T1087.002
Timeframe: Last 7 days
Requires: Falcon Pro+ ; EDIT your VPN CIDRs

Hypothesis: A Type-10 (interactive) logon from a VPN-pool IP followed within 60 minutes by AD enumeration on the same endpoint is the Gentlemen's earliest endpoint-visible chain. Correlation cuts FPs by 10–50× vs. either signal alone.

FP Notes: Legitimate remote admin. Allow-list jump hosts; replace the CIDRs below with your actual VPN-assigned client subnets.

EDIT BEFORE RUNNING: Replace 10.200.0.0/16 and 172.16.50.0/24 with your actual VPN-assigned client pool subnet(s).
↗ Open in Falcon 
// HUNT: The Gentlemen - T1078 + T1087.002 - VPN-pool logon followed by AD recon
// HYPOTHESIS: Interactive logon (LogonType 10) from VPN-assigned client subnet
//             followed by AD enumeration on the same aid within 60 minutes.
// FP RISK: Medium - legitimate remote admin may match
// FP NOTES:
//   - REPLACE the CIDRs below with your actual VPN client pool(s)
//   - Allow-list known jump hosts and remote-admin workstations
// CONFIDENCE: High - correlation tightens specificity
// TIMEFRAME: Last 7 days

// === EDIT THESE CIDRs ===
#event_simpleName = UserLogon
| LogonType = 10
| cidr(RemoteAddressIP4, subnet=["10.200.0.0/16","172.16.50.0/24"])
| logonTime := @timestamp
| logonHost := ComputerName
| logonUser := UserName
| logonSrc  := RemoteAddressIP4
| join(query={
    #event_simpleName = /ProcessRollup2|SyntheticProcessRollup2/
    | in(field=FileName, values=["net.exe","net1.exe","nltest.exe","whoami.exe","dsquery.exe","adfind.exe"], ignoreCase=true)
    | CommandLine = /(Domain Admins|Enterprise Admins|\/dclist|\/domain_trusts|\/groups|\/all)/i
    | reconTime := @timestamp
    | reconCmd  := CommandLine
    | reconFile := FileName
  }, field=[aid], include=[reconTime, reconCmd, reconFile])
| test((reconTime - logonTime) <= 3600000)
| test((reconTime - logonTime) >= 0)
| table([logonTime, logonHost, logonUser, logonSrc, reconFile, reconCmd, reconTime])
| sort(logonTime, order=desc)

Hardening Recommendations

Immediate — Edge Appliance Hardening

Patch first. Confirm current firmware on FortiGate / FortiProxy, SonicWall SMA/NetExtender, and Cisco ASA/FTD against the latest vendor PSIRT advisories. If any device is more than one quarter behind, treat it as compromised until proven otherwise — review config drift, local account creation, and IPSec/SSL VPN logs for the past 90 days.
  • Force MFA on all VPN authentication — preferably hardware/FIDO2; SMS is no longer adequate against session-cookie theft from compromised appliances.
  • Disable SSL VPN on appliances that don't strictly need it (FortiGates that are LAN routers only).
  • Restrict appliance admin UI to a management VLAN only — never expose /remote/login, /admin, or ASDM to the internet.
  • Rotate all local VPN-appliance credentials and IPSec PSKs after any patch.
  • Enable detailed audit logging on the appliance and forward to your SIEM with at least 90-day retention.

Detect — Identity & Endpoint Telemetry

  • Turn on Falcon Identity Protection if licensed — Gentlemen's recon-by-built-in-binaries chain lights up identity policies immediately.
  • Set PowerShell ScriptBlock logging (Event ID 4104) and Module logging via GPO; forward to LogScale.
  • Enable Sysmon or equivalent on Tier 0 / DC hosts to backstop CrowdStrike during sensor downtime.
  • Audit policy: turn on "Audit Distribution Group Management", "Audit Security Group Management", and "Audit Sensitive Privilege Use" on DCs.

Strategic — Reduce Attack Surface

  • Application allow-listing for RMM via AppLocker or WDAC: block AnyDesk, ScreenConnect, RustDesk, Atera, etc., unless your sanctioned tool is the exception.
  • LSA Protection (RunAsPPL) + Credential Guard on all Windows endpoints to neuter LSASS dumping.
  • LAPS for unique local admin passwords (or Windows LAPS / Intune Privilege Management).
  • Tier 0 / Tier 1 / Tier 2 admin model — Domain Admin credentials never used on workstations.
  • Disable LLMNR / NBT-NS / WPAD via GPO.
  • Block outbound to common RMM C2 domains at the egress proxy unless explicitly approved (AnyDesk.net, screenconnect.com, atera.com, splashtop.com, rustdesk.com, etc.).
  • Immutable / offline backups — 3-2-1, with at least one copy that cannot be deleted by a Domain Admin credential.
  • Monitor for new local administrator account creation across the fleet (Event ID 4720, 4732).
  • Limit interactive logons from VPN-pool IPs via Authentication Policy Silos / Tier-0 logon restrictions.
