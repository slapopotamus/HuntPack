1.Executive Summary

UNC3753 is a data-theft extortion cluster, active since at least 2022, that has consistently targeted US-based legal, financial, and professional-services firms. Its tradecraft is almost entirely "living off legitimate tools": there is no custom malware to detect, no encryption payload, and traditional AV rarely fires. The attacker's edge is the human element — a phone call from a fake IT helpdesk that walks the victim into a screen-share and an RMM install. From there, the actor enumerates iManage / OneDrive / mapped drives, keyword-searches for tax forms, SSNs, and client agreements, stages them in the user's Downloads or Roaming profile, and exfiltrates via Rclone, WinSCP, or a browser upload to a consumer cloud account. Extortion email lands within ~30 minutes of exit, with a three-day deadline and threats to post to the LEAKEDDATA / business-data-leaks[.]com leak site.

The defining 2026 escalation: when the vishing call fails, associated actors physically enter the office posing as a technician and copy data to USB — a vector that endpoint telemetry alone cannot fully cover and must be paired with physical controls.

Defender priority: The single highest-leverage detection is unauthorized RMM / remote-support tool installation or execution on a workstation that has no business running it (Q1). Pair it with the curl→msiexec install chain (Q2) and Rclone/WinSCP egress (Q4). Because dwell time is often under one hour, these should be near-real-time alerts, not weekly hunts. Prevention beats detection here: block unsanctioned RMM at the application-control layer and enforce out-of-band identity verification for any "IT support" contact.

3.Hunt Brief & Attack Chain

Working hypothesis: An UNC3753 intrusion will manifest on the endpoint as a short, dense burst of activity: a remote-support or RMM process spawning shortly after a Teams/Zoom/Quick Assist session, followed within minutes by file enumeration, staging in the user profile, and an Rclone/WinSCP or SSH egress to an external host — all under a single user context, often on a BYOD or VDI host. 8.CrowdStrike LogScale CQL Hunt Queries

Field names validated against the Falcon data model where standard. Queries touching Device Control (USB) are flagged ⚠ field unverified — validate in tenant because removable-media event/field names vary by module version.

Q1. Unauthorized RMM / remote-support tool execution

CONF: high | FP: medium | COST: low | Requires: process execution

Looks for: execution of any RMM/remote-support agent named in the GTIG + FBI tool lists. Accomplishes: catches the core UNC3753 foothold step — the moment the victim runs the installer the actor talked them into.

Common FP: teamviewer.exe, zoho assist, atera — legitimately deployed by your own MSP/IT. Maintain an allow-list of sanctioned RMM and the hosts/OUs entitled to run it; exclude those aid/OU values.
// HUNT: Unauthorized RMM / remote-support tool execution
// MITRE: T1219, T1204.002
// CONF: high | FP: medium | COST: low
// REQUIRES: process execution telemetry
// FALSE POSITIVES: sanctioned MSP/IT RMM deployments
#event_simpleName=/ProcessRollup2/
| ImageFileName=/\\(anydesk|bomgar|beserver|screenconnect|connectwise|zaservice|za_connect|zohoassist|assist|superops|rustdesk|syncro|syncrosetup|splashtop|sr(service|server)|atera| ageragent|quickassist|teamviewer|logmein|remotepc)[^\\]*\.exe$/i
// TUNING: exclude approved RMM hosts -> | !in(ComputerName, values=["IT-JUMP01","MSP-RMM02"])
| groupBy([ComputerName, UserName, ImageFileName], function=[count(as=execs), collect([CommandLine], limit=3), min(@timestamp, as=firstSeen)])
| sort(firstSeen, order=desc)
| table([firstSeen, ComputerName, UserName, ImageFileName, execs, CommandLine])

Q2. curl → msiexec silent install chain

CONF: high | FP: low | COST: low | Requires: process command line

Looks for: the exact staging pattern observed in UNC3753 sessions — curl … -o *.msi and/or msiexec /i … /quiet pulling a remote installer. Accomplishes: high-fidelity catch of scripted RMM payload delivery (e.g. the "SuperOps.msi" case).

// HUNT: curl-to-msiexec remote MSI install chain
// MITRE: T1059.003, T1105, T1204.002
// CONF: high | FP: low | COST: low
// REQUIRES: process command line
// FALSE POSITIVES: rare; scripted software deployment via curl is uncommon on user endpoints
#event_simpleName=/ProcessRollup2/
| CommandLine=/curl(\.exe)?\s+.*-o\s+["']?[^"'\s]+\.msi/i
   or CommandLine=/msiexec(\.exe)?\s+\/i\s+.*\.msi.*\/q(uiet|n)?/i
| table([@timestamp, ComputerName, UserName, ParentBaseFileName, ImageFileName, CommandLine])
| sort(@timestamp, order=desc)

Q4. Rclone / WinSCP exfiltration tooling

CONF: high | FP: low-medium | COST: low | Requires: process command line

Looks for: Rclone or WinSCP execution — including renamed binaries — by matching tell-tale command-line flags (copy/sync/--config, /upload) as well as image names. Accomplishes: catches the bulk-exfil step even when the actor renames the binary to evade name-based detection.

Common FP: IT/backup teams legitimately running Rclone to cloud. Exclude known backup service accounts/hosts; alert on user-context execution from Downloads/AppData.
// HUNT: Rclone / WinSCP exfil tooling (incl. renamed binaries)
// MITRE: T1567.002, T1048, T1020
// CONF: high | FP: low-medium | COST: low
// REQUIRES: process command line
// FALSE POSITIVES: sanctioned backup/cloud-sync automation
#event_simpleName=/ProcessRollup2/
| ImageFileName=/\\(rclone|winscp|winscp\.com)[^\\]*\.exe$/i
   or CommandLine=/\brclone\b.*\b(copy|sync|move|lsd|--config|--no-check-certificate)\b/i
   or CommandLine=/\bwinscp.*(\/upload|\/command|sftp:\/\/|scp:\/\/)/i
// TUNING: exclude backup service accounts -> | UserName!=/^(svc_backup|veeam)$/i
| table([@timestamp, ComputerName, UserName, ImageFileName, CommandLine])
| sort(@timestamp, order=desc)

Q5. Outbound SSH (port 22) from workstations / VDI

CONF: medium | FP: medium | COST: medium | Requires: network connection telemetry

Looks for: external port-22 connections originating from user endpoints/VDI, joined back to the initiating process. Accomplishes: surfaces SSH/SCP/SFTP egress (WinSCP, Rclone over SFTP) and protocol tunneling that the FBI flag recommends blocking.

Common FP: developers using SSH/SFTP to legitimate hosts (Git, internal Linux). Exclude dev OUs and known internal/SFTP partner ranges; prioritize connections to fresh external IPs with high byte counts.
// HUNT: Outbound SSH/SFTP from user endpoints and VDI
// MITRE: T1048, T1021.004, T1572
// CONF: medium | FP: medium | COST: medium
// REQUIRES: NetworkConnectIP4 telemetry
// FALSE POSITIVES: developer SSH/SFTP, internal admin tooling
#event_simpleName=/NetworkConnectIP4/
| RemotePort=22
| RemoteAddressIP4!=/^(10\.|192\.168\.|172\.(1[6-9]|2[0-9]|3[01])\.)/
// TUNING: exclude approved SFTP partners -> | !cidr(RemoteAddressIP4, subnet="203.0.113.0/24")
| groupBy([ComputerName, ContextBaseFileName, RemoteAddressIP4], function=[count(as=conns), min(@timestamp, as=firstSeen)])
| sort(conns, order=desc)
| table([firstSeen, ComputerName, ContextBaseFileName, RemoteAddressIP4, conns])

12.Hardening — Tiered

Immediate (this week — low compatibility risk)

  • Block unsanctioned RMM/remote-support binaries via WDAC/AppLocker. Why: MITRE M1042 (Disable or Remove Feature/Program) / M1038 (Execution Prevention). What: CIS Controls v8 §2.3 (unauthorized software). Where: WDAC policy or AppLocker Computer Config → Windows Settings → Security Settings → Application Control Policies. Deny: AnyDesk, Bomgar, Zoho Assist, SuperOps, RustDesk, Syncro, Splashtop, Atera, ScreenConnect/ConnectWise, LogMeIn, RemotePC. Verify: see Playbook P1.
  • Disable USB mass-storage read/write on endpoints handling sensitive data. Why: M1042 — neutralizes the in-person USB exfil vector. What: CIS §10.x removable-media control. Where: registry HKLM\SYSTEM\CurrentControlSet\Services\USBSTOR\Start = 4, or GPO Administrative Templates → System → Removable Storage Access → "All Removable Storage classes: Deny all access = Enabled". Verify: Playbook P2.
  • Disable / remove Quick Assist where not required. Why: M1042. Where: Remove-WindowsCapability -Online -Name App.Support.QuickAssist~~~~0.0.1.0. Verify: Playbook P3.

Near term (1–4 weeks — pilot on a ring first)

  • Conditional Access: corporate-managed devices only for VDI/VPN. Why: M1032 (MFA) / M1037 (Filter Network Traffic). What: Entra Conditional Access requiring compliant/Hybrid-joined device for Windows 365 & VPN. Blocks the BYOD→VDI pivot.
  • Egress control on port 22 + cloud-upload destinations. Why: M1037. What: Block outbound TCP/22 from user/VDI subnets at the firewall; alert on uploads to consumer file-sharing APIs. Enable full session logging with bytes transferred (per FBI FLASH).
  • Phishing-resistant MFA on critical data repositories (iManage, SharePoint, M365). Why: M1032. What: FIDO2/WebAuthn; step-up MFA when accessing VDI nodes or document stores.
  • Real-time alerting in iManage / SharePoint / M365 for rapid keyword searches, search-term spikes, and mass downloads. Why: M1047 (Audit) / M1018. What: threshold alerts per user/session.

Strategic (1–3 months — process / architectural)

  • Out-of-band IT-support verification process. Why: M1017 (User Training). What: publish and train a documented policy for how IT contacts employees and how staff verify a "helpdesk" caller via a known internal channel/callback before granting any remote access.
  • Physical visitor verification & escort policy. Why: M1017 / facilities control. What: photo-ID copy + log, verify technicians against pre-scheduled work orders with the parent org, mandatory supervisor escort for all service personnel.
  • Network segmentation of VDI from crown-jewel repositories. Why: M1030 (Network Segmentation). What: restrict which VDI nodes can reach iManage/file servers; least-privilege access tiers.

P2. Disable USB mass storage (registry + GPO)

# P2 - Block removable mass storage (PowerShell, run as admin)
# Set USBSTOR to disabled (4 = disabled, 3 = enabled):
Set-ItemProperty -Path 'HKLM:\SYSTEM\CurrentControlSet\Services\USBSTOR' `
  -Name Start -Value 4

# Deny all removable storage classes via policy key:
$rs = 'HKLM:\SOFTWARE\Policies\Microsoft\Windows\RemovableStorageDevices'
New-Item -Path $rs -Force | Out-Null
New-ItemProperty -Path $rs -Name 'Deny_All' -Value 1 -PropertyType DWord -Force

# Verification check (expect Start = 4, Deny_All = 1):
Get-ItemProperty 'HKLM:\SYSTEM\CurrentControlSet\Services\USBSTOR' Start
Get-ItemProperty $rs 'Deny_All'
# GPO equivalent: Administrative Templates > System > Removable Storage Access
#   "All Removable Storage classes: Deny all access" = Enabled
