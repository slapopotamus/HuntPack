Executive Summary

APT28 (aka Fancy Bear, Forest Blizzard, Pawn Storm, UAC-0001 — Russian GRU Unit 26165) is running an active espionage campaign codenamed PRISMEX (Trend Micro) / Operation Neusploit (Zscaler) against Ukrainian government bodies and NATO allies' defense, transport, and logistics organizations across Poland, Slovenia, Turkey, Romania, the Czech Republic, Slovakia, the UAE, and Greece. The chain is notable for two operational traits: 24-hour weaponization of a Microsoft Office 1-day (CVE-2026-21509), and cloud-native C2 over filen.io — a legitimate end-to-end encrypted storage service — which bypasses reputation-based proxy filtering by blending with normal HTTPS traffic to a benign provider.

The multi-stage chain begins with a spear-phishing RTF that triggers CVE-2026-21509 to drop PrismexLoader (PixyNetLoader), which uses PNG steganography (a "Bit Plane Round Robin" algorithm) to reconstruct an in-memory .NET CovenantGrunt implant (PrismexStager). The stager polls 24 filen.io subdomains for encrypted task files (AES-256-GCM, HMAC-SHA256), executes them via reflective .NET assembly loading or PowerShell, and streams output back. A separate Outlook-resident backdoor — NotDoor — is dropped via DLL sideloading through a legitimate OneDrive.exe binary in C:\ProgramData, persisting as a VBA macro at %APPDATA%\Microsoft\Outlook\VbaProject.OTM. Some incidents have also included a destructive wiper component that erases %USERPROFILE%.

Multi-stage Chain (typical)

  1. T+0: RTF or LNK attachment opens in Word/Excel. CVE-2026-21509 fires; Office hosts the loader.
  2. T+0–60s: PrismexSheet macro decodes steganographic payload from SplashScreen.png via "Bit Plane Round Robin"; reflectively loads PrismexLoader (.NET) in memory.
  3. T+60s–5min: Persistence established via COM hijacking (T1546.015) and scheduled task (T1053.005). Decoy document shown to victim.
  4. T+5min+: PrismexStager (CovenantGrunt variant) begins polling filen.io across 24 subdomains for encrypted task files. C2 channel is HTTPS to legitimate cloud provider — bypasses reputation filtering.
  5. T+hours–days: Operators drop NotDoor for redundancy: OneDrive.exe + SSPICLI.dll + tmp7E9C.dll + testtemp.ini staged in C:\ProgramData. Encoded PowerShell copies testtemp.ini to %APPDATA%\Microsoft\Outlook\VbaProject.OTM.
  6. Long-haul: NotDoor monitors inbound email via Application_NewMailEx for trigger words; commands executed via VBA. Registry tampering disables Outlook macro warnings (Level=1) and forces auto-load on boot (LoadMacroProviderOnBoot=1).
  7. Optional impact: CovenantGrunt has been observed running wiper commands erasing %USERPROFILE% (T1485) — dual espionage/sabotage capability.

Endpoint-Observable Indicators (the hunt focus)

  • outlook.exe as parent of cmd.exe, powershell.exe, wscript.exe, cscript.exe, mshta.exe, regsvr32.exe, or rundll32.exe
  • winword.exe / excel.exe hosting child processes after RTF/Excel attachment open (CVE-2026-21509 detonation)
  • File write of VbaProject.OTM to %APPDATA%\Microsoft\Outlook\ by any process other than OUTLOOK.EXE
  • OneDrive.exe running from C:\ProgramData\ (legitimate OneDrive lives under C:\Users\<user>\AppData\Local\Microsoft\OneDrive)
  • OneDrive.exe as parent of powershell.exe -EncodedCommand
  • Registry modification of LoadMacroProviderOnBoot, \Outlook\Security\Level, \Outlook\Options\General\PONT_STRING by any process other than Outlook
  • DNS resolution / HTTPS to *.dnshook.site, webhook.site (NotDoor beacon)
  • HTTPS connections to *.filen.io (24 subdomains: gateway.*, ingest.*, egest.*) from any process other than a sanctioned browser
  • Reflective .NET assembly loads from Office or Outlook process contexts (T1620)
  • Creation of %TEMP%\Test\ directory

Hardening Recommendations

The chain has three natural choke points: (1) the initial Office exploit (CVE-2026-21509), (2) Outlook spawning child processes / DLL sideloading, and (3) outbound traffic to cloud-native C2. The controls below are organized by deployment friction (Immediate / Short-term / Long-term) and every recommendation cites at least one framework (MITRE M-number, CIS Benchmark item, MS Security Baseline GPO path) — per the threat-hardening skill's standard shape.

Patch first. Confirm Office is at or beyond the February 2026 cumulative update that ships the CVE-2026-21509 (and CVE-2026-21513) fixes on every endpoint. Microsoft 365 Apps, Office LTSC 2024, and Office 2021/2019 perpetual all have patches. Endpoints more than one Patch Tuesday behind on Office should be treated as exposed until verified.

Immediate (deploy this week) Tier 1

  • Enable ASR rule D4F940AB-401B-4EFC-AADC-AD5F3C50688A — Block Office apps from creating child processes. Directly blocks the Outlook→script-interpreter pattern (Q1) and the winword/excel→child pattern (Q8). MITRE M1040 · CIS Microsoft Windows 10/11 Enterprise 18.9.47.5.1.4 (L1) · MS Baseline Computer Configuration > Admin Templates > Windows Components > Microsoft Defender Antivirus > Microsoft Defender Exploit Guard > Attack Surface Reduction > Configure Attack Surface Reduction rules
  • Enable ASR rule 26190899-1602-49E8-8B27-EB1D0A1CE869 — Block Office communication apps (Outlook) from creating child processes specifically. Layered with the above, eliminates Outlook child-spawn entirely. MITRE M1040 · CIS 18.9.47.5.1.15 (L1)
  • Enable ASR rule 92E97FA1-2EDF-4476-BDD6-9DD0B4DDDC7B — Block Win32 API calls from Office macros. Closes the API-injection escape that PrismexSheet uses to extract steganographic payload. MITRE M1040 · CIS 18.9.47.5.1.8 (L1)
  • Enable ASR rule 3B576869-A4EC-4529-8536-B80A7769E899 — Block Office apps from creating executable content. Stops the post-decode payload write to disk. MITRE M1040 · CIS 18.9.47.5.1.5 (L1)
  • Enable ASR rule BE9BA2D9-53EA-4CDC-84E5-9B1EEEE46550 — Block executable content from email and webmail. Stops the RTF/LNK chain at delivery for users opening attachments outside of corporate mail clients. MITRE M1040 · CIS 18.9.47.5.1.10 (L1)
  • Block macros from running in Office files from the Internet via GPO. Microsoft 365 Apps enforce this by default since 2022; verify Office 2019/2021/LTSC perpetual installations have it set. MITRE M1054 · MS Baseline User Configuration > Admin Templates > Microsoft Word <ver> > Word Options > Security > Trust Center > Block macros from running in Office files from the Internet = Enabled (and same for Excel, PowerPoint, Outlook, Access, Visio)
  • Set Outlook macro security to "Disable all macros without notification" — registry HKCU\Software\Microsoft\Office\<ver>\Outlook\Security\Level = 4. This makes NotDoor's VbaProject.OTM non-functional even if installed. MITRE M1054 · MS Baseline Outlook > Security > Security Form Settings > Macro Security = Disabled
  • Block *.dnshook.site and webhook.site at DNS resolver and NGFW egress (unless dev teams have documented use cases — then allow-list only those segments). MITRE M1021 · M1037
  • Push the SHA256 hashes from §5 to Falcon IOC Management in Prevent mode after a 7-day audit-mode soak. MITRE M1040 · M1049
  • Confirm Office is fully patched — CVE-2026-21509 and CVE-2026-21513 fixes in the Feb 2026 cumulative update. Run Get-AppXPackage Microsoft.OfficeHub on a sample + check Office click-to-run channel version against the Microsoft baseline. MITRE M1051

Short-term (1–4 weeks, requires piloting) Tier 2

  • Deploy Defender for Office 365 Safe Attachments + Safe Links (or equivalent — Proofpoint TAP, Mimecast). Detonate inbound attachments in a sandbox before delivery; rewrites all URLs for runtime evaluation. APT28's RTF + LNK chain is sandbox-detectable. MITRE M1031 · M1021
  • Conditional Access policy blocking unsanctioned cloud storage in Entra ID. If your org doesn't sanction filen.io, dropbox.com personal, mega.nz, etc., a CA policy that requires compliant device + corporate-managed browser for any of those URLs surfaces unmanaged-cloud-storage use across your fleet — including PrismexStager. MITRE M1037 · M1018
  • AppLocker / WDAC publisher rule that blocks OneDrive.exe from C:\ProgramData\ (legitimate OneDrive lives under %LOCALAPPDATA%\Microsoft\OneDrive). Layered defense against IOA-3. MITRE M1038 · M1045
  • PowerShell ScriptBlock + Module logging enabled via GPO, with logs forwarded to LogScale or your SIEM. NotDoor's encoded PowerShell stage decodes into ScriptBlock log Event ID 4104 — a high-fidelity tripwire even if prevention misses. CIS 18.9.100.1 + 18.9.100.2 (L1) · MS Baseline Admin Templates > Windows Components > Windows PowerShell > Turn on PowerShell Script Block Logging = Enabled
  • Disable PowerShell v2 across the fleet. PSv2 bypasses ScriptBlock/Module logging entirely — attackers love it for exactly that reason. Disable-WindowsOptionalFeature -Online -FeatureName MicrosoftWindowsPowerShellV2,MicrosoftWindowsPowerShellV2Root. MITRE M1042
  • Pilot Constrained Language Mode (CLM) for PowerShell via AppLocker on knowledge-worker tier. Breaks reflective .NET assembly loading (T1620) used by PrismexStager — though pilot carefully, as some legitimate admin scripts rely on full-language mode. MITRE M1038 · M1044
  • Falcon Identity Protection policies (if licensed) covering anomalous interactive logon patterns. APT28's long-haul access via NotDoor produces unusual mailbox-to-endpoint patterns that Identity Protection surfaces. MITRE M1031

Long-term / structural (1–3 months) Tier 3

  • Disable VBA for Office entirely for the knowledge-worker tier that doesn't need it. Set HKCU\Software\Microsoft\Office\<ver>\Common\VBAOff = 1 via GPO. This is the structural fix that makes NotDoor and analogous VBA backdoors fundamentally impossible. Pilot first — gather a finance/data-team carve-out plan with explicit signed-macro exceptions before fleet-wide. MITRE M1042 · MS Baseline Admin Templates > Microsoft Office <ver> > Security Settings > Disable VBA for Office applications = Enabled
  • Full WDAC code integrity policy in audit-then-enforce mode. Only signed, approved binaries can run. This kills the OneDrive-from-ProgramData sideload pattern (and many other LOLBin techniques) structurally. High effort, very high payoff. Pilot on knowledge-worker tier first; servers second. MITRE M1045 · M1038
  • CASB / SSE deployment with a cloud-storage governance policy (Netskope, Zscaler ZIA, Cisco Umbrella SIG, Cloudflare Gateway). Goes beyond per-domain blocking — gives visibility into which cloud-storage services are in actual use and lets you make a sanctioned-list decision based on data, not gut. MITRE M1021 · M1037 · M1057 (Data Loss Prevention)
  • Phishing-resistant MFA (FIDO2 hardware keys) for any account with mailbox access to externally-facing inboxes. APT28's spear-phishing chain ends in credential theft for the next campaign; FIDO2 makes the stolen token useless. MITRE M1032
  • Targeted phishing-resistant training program for users in APT28's target verticals (defense/transport/logistics roles). Standard awareness training tops out around 8–10% click rate; targeted role-based simulation gets it to 1–2%. MITRE M1017
  • Outlook plugin allow-list policy via GPO — only signed, IT-approved Outlook plugins load. NotDoor's VbaProject.OTM isn't a plugin per se, but the same trust framework reduces the broader VBA + plugin attack surface. MITRE M1033 · M1045

  '92E97FA1-2EDF-4476-BDD6-9DD0B4DDDC7B',

for ($i = 0; $i -lt $pref.AttackSurfaceReductionRules_Ids.Count; $i++) {
