Executive Summary

RoguePlanet weaponizes a time-of-check / time-of-use (TOCTOU) race in Microsoft Defender's antimalware engine (MsMpEng.exe / mpengine.dll). The engine validates a file path and then operates on it as two non-atomic steps; an unprivileged user flips a directory to an NTFS junction/symlink (a reparse point) in the gap, so Defender's SYSTEM-level write or remediation lands on an attacker-chosen target — yielding code execution as SYSTEM. The public PoC was released 2026-06-10 by the "Nightmare Eclipse" group and works on fully patched Windows 10 and Windows 11 (including Canary), bypassing Microsoft's mid-May hardening of the mpengine!SysIO* APIs.

Two operating modes are documented. The original RCE variant coerces a victim into opening a crafted .vhd(x) from a remote SMB share, causing Defender to overwrite its own files. After Microsoft's May hardening, the author pivoted to a local privilege escalation variant using local symlink redirection — the higher-probability, in-scope risk for most enterprises, since any commodity malware foothold or low-priv user can attempt it. Predecessor exploits from the same group (BlueHammer / CVE-2026-33825, RedSun) used the identical junction-redirection primitive and have already appeared in live intrusions.

Because the exploit subverts the security product itself, the cleanest detection is behavioral, not signature-based: no legitimate workflow has the Defender engine spawn an interactive shell, or write an executable into System32. Those two facts anchor the highest-fidelity hunts in this pack.

⚠ Defender priority: Deploy application allowlisting (WDAC/AppLocker, enforced) — independently confirmed to block the payload — and stand up the MsMpEng.exe → shell lineage detection (Q1) as a Custom IOA today. There is no patch; treat this as an exposure to mitigate, not remediate.

Hunt Brief & Attack Chain

Working hypothesis: A standard-user process stages a reparse point (junction/symlink) in a user-writable directory and races Defender's file operation so that MsMpEng.exe (SYSTEM) writes/executes into an attacker-chosen path, after which a SYSTEM-integrity shell or payload runs. Evidence: anomalous Defender-engine child processes, SYSTEM writes into protected directories, and (RCE variant) disk-image mounts from remote/SMB locations.

CrowdStrike LogScale CQL Hunt Queries

Field names validated against the Falcon event reference; process hunts include SyntheticProcessRollup2. Validate the Defender helper-exclusion list (Q2) against your platform version before promoting any IOA to Prevent. Do not run the live PoC in production — validate detections in a lab.
Q1 · Defender engine spawning an interactive shell / script host
CONF HIGHFP LOWCOST LOW

Looks for: MsMpEng.exe as the parent of a shell or scripting host. Accomplishes: catches the moment of SYSTEM code execution — the flagship signal, no benign equivalent.

// HUNT: Microsoft Defender engine spawning an interactive shell or script host
// MITRE: T1068, T1059.001, T1059.003, T1548 | CONF: high  FP: low  COST: low
// FP NOTES: Defender legitimately launches MpCmdRun.exe / MsMpEngCP.exe, never interactive shells
#event_simpleName=/ProcessRollup2|SyntheticProcessRollup2/
| ParentBaseFileName=/^MsMpEng\.exe$/i
| FileName=/^(cmd|powershell|powershell_ise|pwsh|conhost|cscript|wscript|mshta|rundll32|regsvr32)\.exe$/i
| table([@timestamp, ComputerName, UserName, ParentBaseFileName, FileName, CommandLine, AuthenticationId, SHA256HashData])
| sort(@timestamp, order=desc)
Q2 · Anomalous child process of the Defender engine (broad net)
CONF HIGHFP LOW-MEDCOST LOW

Looks for: any child of MsMpEng.exe that is not a known Defender helper. Accomplishes: catches the payload binary itself when it isn't a standard shell name. FP: Defender helpers (MpCmdRun.exe, NisSrv.exe) — excluded; validate per platform version.

// HUNT: Anomalous child process of the Microsoft Defender engine
// MITRE: T1068, T1574 | CONF: high  FP: low-med  COST: low
// TUNING: exclude the known Defender helper set below; alert on anything else parented by MsMpEng.exe
#event_simpleName=/ProcessRollup2|SyntheticProcessRollup2/
| ParentBaseFileName=/^MsMpEng\.exe$/i
| FileName!=/^(MpCmdRun|MsMpEngCP|NisSrv|MpDlpService|MsSense|SenseIR|SenseCncProxy|MpDefenderCoreService)\.exe$/i
| table([@timestamp, ComputerName, UserName, FileName, ImageFileName, CommandLine, SHA256HashData])
| sort(@timestamp, order=desc)
Q3 · Defender engine writing a PE into a protected system directory
CONF MEDFP LOWCOST LOW-MED

Looks for: MsMpEng.exe as the writing process for a PE in System32/SysWOW64/Program Files. Accomplishes: catches the redirected SYSTEM write — the outcome of the junction race — even if execution is delayed.

// HUNT: Microsoft Defender engine writing a PE into a protected system directory
// MITRE: T1574, T1068 | CONF: medium  FP: low  COST: low-med
// FP NOTES: Defender writes signatures/platform binaries into its OWN dirs, excluded below
#event_simpleName=/PeFileWritten|NewExecutableWritten/
| ContextBaseFileName=/^MsMpEng\.exe$/i
| TargetFileName=/\\(System32|SysWOW64|Program Files( \(x86\))?)\\/i
| TargetFileName!=/\\(Windows Defender|Microsoft\\Windows Defender|Microsoft\\Windows Security)\\/i
| table([@timestamp, ComputerName, ContextBaseFileName, TargetFileName, SHA256HashData])

Hardening — Tiered

No patch exists. The objective is to make the exploit fail-closed: stop the SYSTEM payload from running, and remove the redirection primitives. Primary surface is the Windows endpoint.

Immediate · this week · low risk
  • Application allowlisting (WDAC / AppLocker), enforce mode — M1038. Independently confirmed (ThreatLocker) to block the RoguePlanet payload. Highest-leverage single control.
  • Disable remote symlink evaluation — M1042 ⚠ best-practice, no formal benchmark. fsutil behavior set SymlinkEvaluation R2L:0 R2R:0 removes the remote-redirect primitive.
  • Block outbound SMB (TCP 445/139) from workstations — M1037 (CIS / host firewall). Kills the remote .vhd(x) fetch path.
  • Confirm Defender platform/engine on current channel + Tamper Protection on — M1054. Watch for out-of-band engine version changes (N2).
Near term · 1–4 weeks · pilot first
  • Block standard-user disk-image mounting — M1042. GPO/registry to disable ISO/VHD auto-mount; restrict the Virtual Disk service to admins. (MS Security Baseline.)
  • Attack Surface Reduction (ASR) rules — M1042. Enable prevalence/age/trusted-list execution block + script-host constraints. (Microsoft ASR guidance.)
  • Remove standing local admin; deploy LAPS — M1026 (CIS). Limits the value of a SYSTEM foothold.
  • PowerShell Constrained Language Mode + script-block logging — M1038 / M1040. Raises cost of the SYSTEM shell; improves Q1 evidence.
Strategic · 1–3 months · architectural
  • Enterprise WDAC signed-only code-integrity policy — M1045 / M1038. Make allowlisting the default posture.
  • HVCI + Credential Guard — M1043. Contain blast radius if SYSTEM is reached.
  • Continuous AV-tamper / config-drift monitoring — M1054. Alert on Defender engine self-modification fleet-wide.

Deployable Playbooks

P1 · Disable remote symlink evaluation + verify

:: Run as administrator (or push via GPO startup script / Intune)
fsutil behavior set SymlinkEvaluation R2L:0 R2R:0
:: Verify (expect: R2L disabled, R2R disabled)
fsutil behavior query SymlinkEvaluation

P2 · Block outbound SMB from workstations (Windows Firewall)

# PowerShell — deploy via GPO/Intune to workstation OU
New-NetFirewallRule -DisplayName "Block Outbound SMB 445 (RoguePlanet)" `
  -Direction Outbound -Action Block -Protocol TCP -RemotePort 445 -Profile Any -Group "Hardening-RoguePlanet"
New-NetFirewallRule -DisplayName "Block Outbound NetBIOS 139 (RoguePlanet)" `
  -Direction Outbound -Action Block -Protocol TCP -RemotePort 139 -Profile Any -Group "Hardening-RoguePlanet"
Get-NetFirewallRule -Group "Hardening-RoguePlanet" | Format-Table DisplayName, Enabled, Direction, Action

P3 · Restrict standard-user disk-image mounting

# PowerShell — neutralize Explorer default-mount verb for disk images (pilot first)
$assoc = @(".iso",".vhd",".vhdx")
foreach ($e in $assoc) { cmd /c "assoc $e=Windows.IsoFile" 2>$null }
# Restrict the Virtual Disk service surface to admins (pilot on a ring)
sc.exe sdset vds "D:(A;;CCLCSWRPWPDTLOCRRC;;;BA)(A;;CCLCSWLOCRRC;;;SY)"

P4 · AppLocker baseline (audit → enforce)

# PowerShell — deploy in Audit, then flip to Enforce. Do NOT carve a blanket
# allow for %TEMP%/%ProgramData% (RoguePlanet payload paths).
Set-AppLockerPolicy -XmlPolicy .\RoguePlanet-AppLocker.xml
Get-AppLockerPolicy -Effective | Select-Object -ExpandProperty RuleCollections
